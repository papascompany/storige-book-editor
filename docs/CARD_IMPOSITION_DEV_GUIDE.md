# 카드 양면 조판(Imposition) 개발 가이드

이 가이드는 명함재단기(Slitter/Cutter) 호환을 위한 워커의 조판 알고리즘 구현 방법과 주의사항을 다룹니다.

## 1. 개요
재단기마다 센서 위치가 다르므로, 워커는 **관리자 설정 기반(Configuration-driven)**으로 마커를 생성해야 합니다.

## 2. 구현 가이드라인

### 2.1 관리자 설정 필드 (Admin UI/DB)
조판 설정(Template Set)에 다음 필드를 추가하고 관리자가 수정할 수 있도록 합니다.
- `use_sensor_marker`: (Boolean) 마커 사용 여부
- `marker_x / marker_y`: (Number, mm) 마커 위치
- `marker_width / marker_height`: (Number, mm) 마커 크기

### 2.2 조판 엔진 구현
- **하드코딩 금지**: 마커의 위치와 크기를 코드 내에 고정하지 마십시오.
- **좌표 변환**: 관리자가 입력한 mm 수치를 PDF 포인트(72dpi)로 정확히 변환하여 드로잉합니다.
- **레이어**: 마커가 다른 인쇄물에 가려지지 않도록 항상 가장 마지막(최상위)에 드로잉합니다.

### 2.3 양면 정합 알고리즘
- 앞면 조판 결과와 뒷면 조판 결과가 가로축을 기준으로 완벽히 대칭(Mirroring)되도록 좌표계를 설계합니다.

## 3. 테스트 케이스
1. **위치 이동 테스트**: `marker_x` 값을 변경했을 때 마커가 수평 이동하는지 확인합니다.
2. **크기 변경 테스트**: `marker_width` 값을 변경했을 때 마커의 가로 길이가 설정값과 일치하는지 확인합니다.
