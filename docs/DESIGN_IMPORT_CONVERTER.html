①개요

@storige/indesign-import 는 디자인 소스(InDesign IDML, Photoshop PSD)를 Storige 템플릿으로 변환하는 패키지다. 결과는 SVG 파일이 아니라 Fabric.js 캔버스 객체 배열(canvasData.objects) 과 (표지 펼침면) spreadConfig 를 담은 DraftTemplateDto 로, 기존 POST /templates 등록 경로에 그대로 투입된다.

  • 브라우저(admin)와 Node 양쪽 동작 — 핵심 src/ 는 Node 내장 API 미사용. 환경 분기(typeof document)로 래스터화 백엔드만 교체(browser <canvas> / Node sharp).
  • DB 직접 쓰기 없음 — 변환기는 CreateTemplateDto 호환 JSON 생성 → 관리자 검수 → 기존 API 등록.
  • 포맷 자동 감지 — 확장자로 IDML/PSD 구분(detectFormat).
입력결과 Template 타입spreadConfig주 용도
IDML (표지 펼침면)type='spread'있음 (5영역)책 표지(앞/뒤/책등/날개)
PSD (단품)type='page' / 'cover'없음(단일 페이지)명함 · 내지 단품 등

③IDML (표지 펼침면)

💡
왜 IDML 인가
.indd 는 독점 바이너리 → Node 단독 파싱 불가. .idml 은 ZIP+XML 공개사양이라 추가 라이선스 없이 파싱. INDD 는 scripts/indd-to-idml.jsx(InDesign ExtendScript)로 IDML 내보낸 뒤 입력하는 옵션 경로(InDesign Server 시 무인 변환 가능).

파싱 규약 (src/idml/reader.mjs)

IDML 파일추출 내용
Spreads/Spread_*.xml페이지 기하 + 페이지아이템 geometry(z-순서)
Stories/Story_*.xml텍스트 내용 + 폰트/크기/색
Resources/Graphic.xml색상(CMYK/RGB/별색)
Resources/Fonts.xml폰트 패밀리 목록
Resources/Preferences.xml블리드(DocumentBleedTopOffset)
  • 문서순(z-순서) 보존 — preserveOrder 파서로 문서 순서대로 수집. 안 지키면 책등 패널 아래 도형이 위로 올라옴. 그룹은 재귀로 펼치며 부모 ItemTransform 합성.
  • 줄바꿈(Br) 보존 — Content=텍스트, Br=\n. 단락 사이에도 \n → 다단 텍스트 영역 이탈 방지.
  • 베지어 경로 복원 — Anchor·LeftDirection·RightDirection → 직선(L)/3차 베지어(C)/닫힘(Z) (src/geometry/path.mjs).
  • CMYK 원본 보존 — sRGB 근사 hex + 원본 4도값을 cmykFill 로 보존.
  • 별색(Spot) 감지 — 판정은 반드시 @_Model(=Spot/Mixed Ink) 기준. Spot 도 Space=CMYK 인 경우가 많아 Space 판정은 놓침.

표지 5영역 (spreadConfig)

좌→우 5영역(src/geometry/regions.mjs, canvas-core SpreadLayoutEngine 규약과 동일):

back-wing뒤날개
back-cover뒤표지
spine책등(가변)
front-cover앞표지
front-wing앞날개

총폭(mm) = wing×2(활성 시) + cover×2 + spine, 0.1mm 반올림.

  • 날개 판정 — 3페이지=[cover, spine, cover](날개 없음), 5페이지=[wing, cover, spine, cover, wing](날개).
  • 책등 추정 — 가장 좁은 페이지=책등. 표지폭=책등 제외 최대폭, 날개폭=최소폭(좌우 동일 가정).
  • 영역/앵커 — 객체 중심 x 를 resolveRegionAtX(x ≥ r.x && x < r.x + r.width) 에 통과 → regionRef + 정규화 앵커(xNorm) 부여(런타임 책등 가변 입력).
📐
좌표 변환
IDML pt(1/72in) → mm → 캔버스 px(DPI 150). 상수 src/geometry/units.mjs(canvas-core math.ts 와 동일). 아핀 변환은 ItemTransform([a,b,c,d,tx,ty]) 합성/적용/분해. 행렬식 음수(det<0)는 한 축 반전(flip)으로 scaleY 부호에 흡수. 경로형은 회전이 좌표에 반영 → angle=0. 책등 세로쓰기 TextFrame 은 프레임 치수 유지 + angle=90°.

두 변환 모드: 벡터 vs 하이브리드

벡터 (mode='vector', 기본)
  • 도형/배경 → 편집 가능 벡터(rect/ellipse/path)
  • 텍스트 → 편집 가능 textbox
  • 적합: 단순한 디자인(정밀 편집)
  • 손실: 무손실(벡터)
하이브리드 (mode='hybrid')
  • 비텍스트 전체 → 300dpi PNG 1장(최하단)
  • 텍스트 → 편집 가능 textbox(이미지 위)
  • 적합: 효과·그라디언트·별색이 많은 디자인
  • 손실: 비텍스트는 PNG 해상도 의존

하이브리드는 rasterizeArtwork(dto,{dpi:300})(src/raster/rasterize.mjs)가 비텍스트만 그린 SVG 를 물리치수(totalWidthMm/HeightMm) 기준 목표 픽셀로 래스터화(텍스트 제외·투명 배경) → 최하단 이미지로 깔고 텍스트만 위에 둔다.

생성되는 객체 속성

  • 안정적 id: idml-{Self} (추적/선택/잠금), selectable/evented:true, isUserAdded:false.
  • 중심 좌표 규약: left/top = 중심(originX/originY='center').
  • cmykFill(원본 4도, 출력단 미사용), spotColor(별색명), fillRule:'evenodd'(컴파운드 패스 구멍 보존).
  • meta:{regionRef, anchor} — 런타임 책등 가변 재배치 입력. _idml 디버그 필드는 저장 전 제거.

④PSD (단품)

⚠️
텍스트 이중 렌더 방지
비텍스트 레이어를 합성한 배경 PNG 에는 텍스트 레이어를 반드시 제외해야 한다(kind==='raster' 만 전달). 그래야 "추출 텍스트 제외 합성"이 되어 텍스트가 두 번 찍히지 않는다.

파싱 (src/psd/reader.mjs)

  • 텍스트 레이어 → 편집 가능 textbox — 내용 + 근사 폰트/크기/색(EngineData best-effort, 관리자 확정 전제). 픽셀 bbox 0 이어도(합성 PSD 흔함) 폰트/내용으로 근사 박스.
  • 비텍스트 레이어 → 배경 PNG 1장 — kind:'raster' 만 합성.
  • z-순서 뒤집기 — @webtoon/psd children 은 위→아래([0]=최상단). 합성/렌더는 아래부터라야 하므로 뒤집어 bottom→top(배경 맨 아래, 텍스트 위).
  • DPI 추출 — resolutionInfo DPI 로 px→mm(PixelsPerCM 이면 ×2.54). 정보 없으면 72dpi 가정.
  • 줄바꿈 정규화 — Photoshop \r → \n.

배경 합성 (src/psd/rasterizePsd.mjs)

  • PSD 원본 px 해상도 그대로 합성(인쇄 해상도 보존).
  • 브라우저: 레이어별 임시 <canvas> → drawImage 알파 합성(위치/클리핑 자동).
  • Node: sharp 합성. ⚠️ 캔버스 밖 레이어 스킵(sharp 음수/초과 오프셋 거부). 브라우저는 drawImage 로 정상 클리핑.

단일 페이지 템플릿 (src/convert/toSinglePageTemplate.mjs)

  • 좌표: PSD px → 물리 mm(소스 dpi) → 캔버스 px(템플릿 dpi 150). 스케일 = 150 / sourceDpi.
  • 배경 아트워크(최하단) + 텍스트 textbox(근사값).
  • 출력 = type='page' 또는 'cover' 단일 페이지 Template (spreadConfig 없음).
  • 경고: 폰트 확정 필요 · "텍스트 폰트/크기/효과는 추출 근사값 — 에디터에서 관리자 확정 필요" · (CMYK PSD) 지원 제한 · 해상도 <300dpi.

⑤책등 가변 (런타임 파생)

표지 spread 템플릿의 spineWidthMm 는 권위 고정값이 아니라 런타임 파생값이다. 권위 검증 validateSpreadAgainstAuthority(packages/types/src/index.ts)는 cover 가로/세로, 날개 유무/폭 4필드만 비교하고 책등·총폭은 비교하지 않는다.

책등폭 공식 — packages/types/src/index.ts
spineWidth(mm) = (pageCount / 2) × paperThickness + bindingMargin
  • 셋(editorMode='book')이 구성되면 내지 수 변동 시 SpreadPlugin.resizeSpine(canvas-core)가 책등 자동 리사이즈 + meta.regionRef/anchor 기준 영역 객체 재배치.
  • 표지 IDML 단독으론 책 셋 완전 자동화 불가 — 책모드 검증이 SPREAD 1 + PAGE≥1 강제(내지 필수).
⚠️
구현 주의 (meta 라운드트립)
객체 meta(regionRef/anchor)는 resizeSpine 재배치의 유일한 입력인데, 표준 단일 Template 저장경로 직렬화 화이트리스트에 meta 포함 여부 확인 필요(useTemplateSetSave 경로만 포함). 탈락 시 책등 가변이 깨질 수 있음. 보완: 화이트리스트에 'meta' 추가, 또는 spreadConfig.regions 기반 on-load regionRef 재계산.

📚

seedTemplateId 를 받은 TemplateSetForm 이 자동으로: 셋 타입 BOOK · editorMode=BOOK · 셋 판형 = 표지 한 면 크기(spreadConfig.spec.coverWidth/Height) · 표지(spread)를 첫 템플릿(필수)으로 자동 추가 · canAddPage=true. 관리자는 내지(page) 추가 + 페이지 수 범위(pageCountRange) 만 수동으로 채운다. 상품 연결(ProductTemplateSet, sortcode)은 별도 수동 단계.
