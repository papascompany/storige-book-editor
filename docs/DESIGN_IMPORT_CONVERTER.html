①개요

@storige/indesign-import 는 디자인 소스(InDesign IDML, Photoshop PSD)를 Storige 템플릿으로 변환하는 패키지다. 결과는 SVG 파일이 아니라 Fabric.js 캔버스 객체 배열(canvasData.objects) 과 (표지 펼침면) spreadConfig 를 담은 DraftTemplateDto 로, 기존 POST /templates 등록 경로에 그대로 투입된다.

  • 브라우저(admin)와 Node 양쪽 동작 — 핵심 src/ 는 Node 내장 API 미사용. 환경 분기(typeof document)로 래스터화 백엔드만 교체(browser <canvas> / Node sharp).
  • DB 직접 쓰기 없음 — 변환기는 CreateTemplateDto 호환 JSON 생성 → 관리자 검수 → 기존 API 등록.
  • 포맷 자동 감지 — 확장자로 IDML/PSD 구분(detectFormat).
입력결과 Template 타입spreadConfig주 용도

③IDML (표지 펼침면)

💡
왜 IDML 인가
.indd 는 독점 바이너리 → Node 단독 파싱 불가. .idml 은 ZIP+XML 공개사양이라 추가 라이선스 없이 파싱. INDD 는 scripts/indd-to-idml.jsx(InDesign ExtendScript)로 IDML 내보낸 뒤 입력하는 옵션 경로(InDesign Server 시 무인 변환 가능).

파싱 규약 (src/idml/reader.mjs)

IDML 파일추출 내용
Spreads/Spread_*.xml페이지 기하 + 페이지아이템 geometry(z-순서)
Stories/Story_*.xml텍스트 내용 + 폰트/크기/색
Resources/Graphic.xml색상(CMYK/RGB/별색)
Resources/Fonts.xml폰트 패밀리 목록
Resources/Preferences.xml블리드(DocumentBleedTopOffset)
  • 문서순(z-순서) 보존 — preserveOrder 파서로 문서 순서대로 수집. 안 지키면 책등 패널 아래 도형이 위로 올라옴. 그룹은 재귀로 펼치며 부모 ItemTransform 합성.
  • 줄바꿈(Br) 보존 — Content=텍스트, Br=\n. 단락 사이에도 \n → 다단 텍스트 영역 이탈 방지.
  • 베지어 경로 복원 — Anchor·LeftDirection·RightDirection → 직선(L)/3차 베지어(C)/닫힘(Z) (src/geometry/path.mjs).
  • CMYK 원본 보존 — sRGB 근사 hex + 원본 4도값을 cmykFill 로 보존.
  • 별색(Spot) 감지 — 판정은 반드시 @_Model(=Spot/Mixed Ink) 기준. Spot 도 Space=CMYK 인 경우가 많아 Space 판정은 놓침.

표지 5영역 (spreadConfig)

좌→우 5영역(src/geometry/regions.mjs, canvas-core SpreadLayoutEngine 규약과 동일):

back-wing뒤날개
back-cover뒤표지
spine책등(가변)
front-cover앞표지
front-wing앞날개

총폭(mm) = wing×2(활성 시) + cover×2 + spine, 0.1mm 반올림.

  • 날개 판정 — 3페이지=[cover, spine, cover](날개 없음), 5페이지=[wing, cover, spine, cover, wing](날개).
  • 책등 추정 — 가장 좁은 페이지=책등. 표지폭=책등 제외 최대폭, 날개폭=최소폭(좌우 동일 가정).
  • 영역/앵커 — 객체 중심 x 를 resolveRegionAtX(x ≥ r.x && x < r.x + r.width) 에 통과 → regionRef + 정규화 앵커(xNorm) 부여(런타임 책등 가변 입력).
📐
좌표 변환 — content↔scene(중앙원점)
IDML pt(1/72in) → mm → content px(좌상단원점, DPI 150) → scene px(중앙원점, 객체 left/top). 상수 src/geometry/units.mjs(canvas-core math.ts 와 동일). 아핀 변환은 ItemTransform([a,b,c,d,tx,ty]) 합성/적용/분해. 행렬식 음수(det<0)는 한 축 반전(flip)으로 scaleY 부호에 흡수. 경로형은 회전이 좌표에 반영 → angle=0. 책등 세로쓰기 TextFrame 은 프레임 치수 유지 + angle=90°.
🧭 content↔scene 변환은 COORDINATE_SYSTEM.md 가 정본 — 부호 규약은 src/geometry/centerOrigin.mjs(SSOT) 한 곳에만 두고, 변환기·미리보기·래스터 4파일이 모두 이 헬퍼만 쓴다(드리프트 방지). 회귀 가드: centerOrigin.test.mjs + preview/renderInvariants.test.mjs.

세 변환 모드: 벡터 / 하이브리드 / flat-spine 2026-06-12

각 모드는 spreadConfig.conversionMode 와 1:1 대응(vector→full, hybrid→flat-spread, flat-spine→flat-spine). 설계 결정·함정은 ⑬ 참조.

벡터 (mode='vector', 기본) → full
  • 도형/배경 → 편집 가능 벡터(rect/ellipse/path)
  • 텍스트 → 편집 가능 textbox
  • 책등 가변: 허용(meta 재배치)
  • 적합: 단순한 디자인(정밀 편집) · 손실: 무손실
하이브리드 (mode='hybrid') → flat-spread
  • 비텍스트 전체 → 300dpi PNG 1장(최하단)
  • 텍스트 → 편집 가능 textbox(이미지 위)
  • 책등 가변: 차단(책등 고정)
  • 적합: 효과 많은 디자인 + 책등 고정 상품 · 손실: PNG 해상도 의존
flat-spine (mode='flat-spine') → flat-spine
  • 전폭 300dpi 1회 렌더 → 3크롭(back / spine 3배폭 / front, 흰 배경 합성 불투명)
  • 텍스트 → 편집 가능 textbox(이미지 위)
  • 책등 가변: 허용 — spine PNG 3배폭이 두께 변동 흡수
  • 적합: 효과 많은 디자인 + 책등 가변 책 셋 · 손실: PNG 해상도 의존

하이브리드는 rasterizeArtwork(dto,{dpi:300})(src/raster/rasterize.mjs)가 비텍스트만 그린 SVG 를 물리치수(totalWidthMm/HeightMm) 기준 목표 픽셀로 래스터화(텍스트 제외·투명 배경) → 최하단 이미지(고정 — 아래 잠금)로 깔고 텍스트만 위에 둔다. flat-spine 은 같은 전폭 래스터를 computeFlatSpineCrops(convert/flatSpineGeometry.mjs, 순수 함수) 경계로 3크롭(raster/cropArtwork.mjs, 흰 배경 합성)해 spine-artwork(최하단, 3배폭) → back-artwork → front-artwork → 텍스트 z-order 로 깐다 — back/front 가 spine 겹침 구간을 불투명하게 은폐하고, 책등이 늘어나면 가려졌던 spine PNG 가 드러난다.

미리보기·래스터 좌표 주의 (③④ 회귀 지점)
preview/svg.mjs·raster/rasterize.mjs 는 SVG viewBox="0 0 W H"(content 좌상단원점)인데 객체 left/top 은 scene(중앙원점)이다 → 비-path 객체는 sceneToContent(centerOrigin.mjs SSOT)로 환산해야 region·path 와 정합. 누락 시 back-cover(−x) 객체가 viewBox 밖 클립·front 좌측 변위(527b85b 수정, a64d409 단일화+불변식 가드). path 의 d 는 이미 content 절대 px → 변환 제외.

생성되는 객체 속성

  • 안정적 id: idml-{Self} (추적/선택/잠금), selectable/evented:true, isUserAdded:false. (배경 아트워크 예외 — 아래 잠금)
  • 중심 좌표 규약: left/top = scene(중앙원점, originX/originY='center'). content→scene 변환 geometry/centerOrigin.mjs(SSOT).
  • styles:{} + per-run styles A2+A3 — textbox 필수(fabric 5.5 키 누락 시 stylesToArray 저장 크래시·무한로딩, 9628f1a). 혼합 서식은 convert/textStyles.mjs(SSOT)가 객체 base(문자수 가중 지배값)와 다른 속성만 diff(전 run 동일이면 {}). 객체 단위: charSpacing=Tracking 1:1(1/1000em 동단위) · lineHeight=LeadingPt/(1.13×pt)(auto=120%→≈1.0619) · textAlign=Justification · FontStyle명→fontWeight(SemiBold=600·Heavy=900). 세로짜기는 단일 스타일 + 자간→lineHeight 환산((1+trk/1000)/1.13). 혼합/장평은 지배값 근사+경고.
  • rx/ry — Oval 은 반경 명시(fabric.Ellipse 는 width/height 가 아닌 rx/ry; 누락 시 비가시·width:0 박제, 527b85b). Rectangle 균일 RoundedCorner 도 rx/ry A6 — extractCorner(per-corner 4값+legacy, option≠None ∧ radius>0 게이트로 잔존 기본값 12.7pt 오염 차단), |scale| 베이크, 비클램프(pill 형상 보존). 한 축 0 라운딩은 직각+경고(fabric _initRxRy 0=미설정 분기 차단). 고아 strokeWidth(색 없는 weight)는 제거.
  • 그라디언트 fill A1 — fabric Gradient plain object({type, coords, colorStops, gradientUnits:'pixels'}, _initGradient 자동 부활·왕복 안전). 끝점 E 를 inner pt 공간에서 합성 후 S 와 동일 SSOT 매퍼로 사상 → 회전/플립/스케일(베이크 포함) 자동 정합. flipY 미러 보정+경고, Midpoint≠50 은 50% 혼합 중간 스톱. FLAT 경로는 render/svgGradient.mjs 공통 defs(objectBoundingBox — 비정사각 대각 각도 근사 경고). 텍스트 그라디언트는 검정 대체+경고(보류).
  • placed 이미지 A5 — 배치 SSOT = inner Image ItemTransform(FFO crop 은 보조 — 단독 신뢰 금지). linkedImages 매칭 시 프레임 가시영역을 픽셀 크롭 베이크(browser canvas/Node sharp, JPEG q90)한 plain image 로 동일 z-order 인덱스 치환 — cropX/clipPath 불사용(직렬화 함정). FULL=편집 가능 이미지(id 승계), FLAT=래스터에 베이크. 미매칭은 '동반 업로드 매칭 실패' 구분 경고.
  • cmykFill(원본 4도, 출력단 미사용), spotColor(별색명), fillRule:'evenodd'(컴파운드 패스 구멍 보존).
  • meta:{regionRef, anchor} — 런타임 책등 가변 재배치 입력. flat 아트워크는 meta.flatArtwork('spine'|'back'|'front') 추가(⑬). _idml 디버그 필드는 저장 전 제거.

④PSD (단품)

⚠️
텍스트 이중 렌더 방지

⑬템플릿 유형 3종 — conversionMode (2026-06-12)

표지 spread 템플릿을 변환 방식 기준 3유형으로 체계화(7585e38). 모드별 동작 비교는 ③(세 변환 모드), admin UI 문구는 IDML_IMPORT_FLOW.md §1[3].

설계 결정

  • spreadConfig.conversionMode 스탬프 — 'full' | 'flat-spread' | 'flat-spine'(types SpreadConversionMode). vector→full, hybrid→flat-spread, flat-spine→flat-spine. JSON 필드라 DB 마이그레이션 불필요, 레거시(미존재)=full 간주.
  • z-order 은폐(flat-spine) — spine-artwork(책등 중심 3배폭 크롭, 최하단) 위를 back/front-artwork 가 덮음. 책등이 늘면 가려졌던 spine PNG 노출. 전제: 3장 모두 흰 배경 합성 불투명(투명이면 아래층이 비쳐 이중상).
  • clipPath 절대 금지 — 직렬화 유실 함정. 크롭은 전부 픽셀 베이크(flat 3크롭·placed 공통 규약).
  • 앵커 규약 — back/front 는 region anchor(regionRef+xNorm), spine-artwork 는 canvas anchor(content 중앙 규약 = scene (0,0)). left 는 가정값 0 이 아닌 크롭 실제 중심(centerPx)에서 유도(클램프/반올림 퇴화 자동 흡수).
  • meta.flatArtwork('spine'|'back'|'front') — 런타임 재배치 분기 식별자. repositionObjects 가 spine-artwork 를 무이동·무스케일 명시 가드.
  • 크롭 지오메트리 단일 출처 — computeFlatSpineCrops(순수 함수): 경계 px 는 누적 mm 를 한 번씩만 반올림 → back+spineBand+front = 전폭 항상 성립. rasterizeArtwork 와 동일 공식, 불일치 시 명시 throw.

편집기/canvas-core 가드 · admin/API

  • flat-spread = 책등 고정 — editor spineCalculator 단일 가드가 재계산을 정상 스킵 + SpreadPlugin.resizeSpine 방어적 no-op(이중 방어). 통짜 PNG 라 가변 시 디자인이 찢어지기 때문.
  • flat-spine = 책등 가변 허용 — spine-artwork 만 불변, back/front 는 region anchor 로 평행이동.
  • admin 모드 3종 UI(TemplateImport.tsx) — '벡터 (전체 편집형)' / '펼침면 플랫형 (책등 고정)' / '책등가변 3분할 플랫형'. flat-spine 은 3장 dataURL 각각 /storage 업로드.
  • API 검증(validateAndNormalizeSpreadConfig) — conversionMode 화이트리스트 + flat-spine 은 back-cover/spine/front-cover regions 필수(kind/position 키 둘 다 허용). update 시 conversionMode 보존 병합(누락 시 기존 값 유지 — 'full' 강등 라운드트립 유실 차단). TemplateEditorView 저장 시에도 보존.
