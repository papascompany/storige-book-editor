# P2-7 Admin Playwright E2E 확장 보고서 (2026-05-04)

## 요약

기존 Admin Playwright는 smoke 시나리오(로그인 + 사이드바) 2개 spec / 6개 테스트만 존재. P2-7 작업으로 다음 3개 신규 시나리오 추가:

1. **Worker 테스트 페이지** (`/worker-test`) — 폼 가시성 + URL 입력 + 잡 생성 발사
2. **큐 모니터 위젯** (Dashboard) — 5초 폴링 동작 + 상태 태그
3. **라이브러리 카테고리 필터** (`/library/backgrounds`) — Select 변경 → query param 재발사

총 spec 5개 / 테스트 15개.

## 신규 시나리오 상세

### 1. Worker Test (`worker-test/pdf-validation-flow.spec.ts`)
- `worker-test page renders with validation form` — 페이지 제목 + Tabs (URL/Upload) 가시성
- `URL input tab accepts a storage path` — placeholder 매칭 + fill 동작
- `submit triggers POST /worker-jobs/validate (dry-run safe)` — 실 잡 생성, 운영에서는 자동 skip

### 2. Queue Monitor (`queue-monitor/widget-polling.spec.ts`)
- `widget renders with queue cards` — "워커 큐 모니터링" 카드 + 3개 큐 라벨
- `refetches /health/queues at 5s interval` — 네트워크 가로채기로 6초 후 ≥2회 호출 확인 + 간격 검증 (3-8s)
- `shows status tag (정상 / 주의 / 적체)` — 상태 태그 셋 중 하나 노출

### 3. Library Filter (`library/category-filter.spec.ts`)
- `backgrounds page shows category filter` — 페이지 제목 + Select placeholder 가시성
- `selecting category triggers query with category param` — 옵션 선택 → `category=` 파라미터 포함 GET 발사
- `clearing filter removes category param` — Clear 버튼 → 파라미터 제거된 GET 재발사

## 안전 정책 (운영/로컬 호환)

- **E2E_ADMIN_PASSWORD env 미설정**: 로그인 의존 테스트 자동 skip (login-page-renders 단일 테스트만 실행)
- **E2E_BASE_URL=https://admin.papascompany.co.kr**: 운영 환경 read-only 검증
- **E2E_DRY_RUN=true** 또는 운영 URL 자동 감지: Worker 잡 생성 단계 skip (운영 데이터 무영향)

## 셀렉터 강건성

Ant Design `<Input>`은 default `type="text"`. 안정적 매칭을 위해 다단계 셀렉터:
- 이메일: `input[autocomplete="email"], input[placeholder="이메일"], input[type="email"]`
- 비밀번호: `input[autocomplete="current-password"], input[placeholder="비밀번호"], input[type="password"]`

## 실행 방법

```bash
# 운영 admin (read-only, dry-run)
cd apps/admin
E2E_BASE_URL=https://admin.papascompany.co.kr \
  E2E_ADMIN_PASSWORD='<CLAUDE.local.md §5 STORIGE_ADMIN_PASSWORD>' \
  E2E_DRY_RUN=true \
  pnpm test:e2e

# 로컬 dev server (자동 기동)
E2E_ADMIN_PASSWORD='<...>' pnpm test:e2e

# UI 모드 (디버깅)
pnpm test:e2e:ui
```

## 학습 / 셀렉터 가이드

이번 작업으로 운영 admin Antd UI에 대한 Playwright 셀렉터 패턴 정립:

| Antd 컴포넌트 | ❌ 동작 안 함 | ✅ 권장 |
|---|---|---|
| `<Input>` (Form.Item) | `input[type="email"]` | `input[autocomplete="email"], input[placeholder="이메일"]` |
| `<Title level={2}>` | `getByText('대시보드')` | `getByRole('heading', { name: '대시보드' })` |
| `<Select>` placeholder | `getByPlaceholder('카테고리 필터')` | `.ant-select-selection-placeholder { hasText: '카테고리 필터' }` |
| `<Select>` 자체 클릭 | — | `.ant-select.filter({ hasText: '카테고리 필터' })` |
| 페이지 로드 대기 | `waitForLoadState('networkidle')` (Dashboard 5초 폴링 때문에 영원히 대기) | `waitForLoadState('domcontentloaded')` 또는 명시적 element wait |
