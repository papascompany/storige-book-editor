①템플릿 등록 방법

템플릿을 만드는 길은 3가지입니다. 진입은 템플릿 > 템플릿 관리(/templates) 우상단의 두 버튼.

1-A

IDML 변환

InDesign 표지 펼침면(.idml)을 type=spread 템플릿으로. 케이스별 3가지 변환 방식.

버튼: IDML 가져오기 → /templates/import
1-B

PSD 변환

Photoshop 단품(.psd)을 단일 페이지(page/cover)로. 같은 화면에서 자동 감지.

버튼: IDML 가져오기 → /templates/import
1-C

직접 수기 등록

빈 캔버스에서 객체를 직접 배치. 모든 타입 가능.

버튼: 템플릿 생성 → /templates/editor

1-A. IDML 변환 등록 (케이스별)

InDesign에서 만든 표지 펼침면(날개+앞표지+책등+뒤표지+날개 5영역)을 변환합니다.

사전 준비 (InDesign)

  • InDesign → 파일 > 다른 이름으로 저장 > InDesign Markup (IDML). (.indd는 scripts/indd-to-idml.jsx로 변환)
  • ⚠️ 폰트는 IDML에 임베드되지 않음 → 변환 화면 폰트 매칭에서 시딩 필요.
  • ⚠️ 배치(placed) 이미지는 픽셀이 없음 → 원본 이미지를 같은 드롭에 함께 올리면 자동 복원.

[3] 미리보기·매칭·경고 검수

  • SVG 미리보기 — z-순서, 책등 세로글씨(90°), 줄바꿈, 베지어 경로 육안 확인.
  • 감지된 사양 — 표지 한 면(coverWidth)·높이·책등(spineWidth 가변)·날개·재단여백·총폭.
  • 배치 이미지 매칭 — ✓ 복원 참조 없음 ✗ 실패
  • 폰트 매칭(라이브러리) — ✓ 사용 가능 비활성 → 활성화 ✗ 미등록 → 폰트 파일 등록(TTF/OTF/WOFF2)
  • 검수 경고 — 별색·풀블리드·자간/정렬 혼합(지배값 근사)·장평(미적용)·미해석 색(검정 대체)·텍스트 그라디언트(검정 대체)·세로쓰기 등.

1-B. PSD 변환 등록

Photoshop 단품(명함·내지 단품)을 단일 페이지 템플릿으로. IDML과 같은 화면(/templates/import)에서 .psd 업로드 시 자동 감지.

항목IDMLPSD
산출물펼침면(spread)단일 페이지(page/cover)
spreadConfig있음(5영역·책등 가변)없음
변환 방식 토글3종없음(텍스트 + 배경 PNG 고정)
배치 이미지 동반사용미사용(픽셀 내장)
  1. 업로드.psd 업로드 → 자동 PSD 모드.
  2. 미리보기 + 감지된 사양판형 가로×세로 · 편집 텍스트 레이어 수 · 배경 합성 레이어 → 배경 PNG 1장.
  3. 페이지 타입 선택내지/단품 (page) = type=page / 표지 (cover) = type=cover.
  4. 저장템플릿명·카테고리 → [템플릿으로 저장] → 편집기로 이동해 폰트·크기·효과 확정.

1-C. 직접 수기 등록 (디자인 요소 수기 배치)

빈 캔버스에서 직접 객체를 배치. 변환 소스가 없거나 단순 양식/배경/도형 위주 템플릿에 적합.

  1. 템플릿 생성 클릭템플릿 관리 → [템플릿 생성] → "새 템플릿 설정" 모달.
  2. 이름 입력
  3. 템플릿 타입 선택내지(Page) / 표지(Cover) / 책등(Spine) / 날개(Wing) / 스프레드(Spread, 책모드용) / 면지(EndPaper).
  4. 치수 입력일반 타입: 가로/세로(mm, 10~1000). 스프레드: "스프레드 템플릿 설정"에서 표지 한 면·날개·초기 책등(셋에서 가변).
  5. 편집기에서 디자인편집기(iframe)에서 텍스트·이미지·요소(SVG·도형)·배경·프레임·QR 등 직접 배치.
  6. 저장편집기 저장 → admin이 TEMPLATE_SAVED 수신 → "템플릿이 저장되었습니다."
수기 등록 템플릿도 변환 결과와 동일하게 템플릿셋에 넣고 상품에 연결할 수 있습니다.

1) 배경 아트워크 고정 — ARTWORK_LOCK

플랫형(hybrid/flat-spine)·PSD 배경 PNG는 변환기가 ARTWORK_LOCK으로 emit:

  • 고정(selectable:false) — 이동/삭제 불가
  • 패널 숨김 — 레이어 패널 미노출
  • 관리자 잠금 — 템플릿 요소로 보존

효과: 고객은 텍스트만 편집, 배경 디자인은 흔들리지 않음.

2) 객체별 편집 권한 — movable/deleteable

현재 정책 default-permissive (PERM-1):

  • 플래그 미지정(undefined) = 허용
  • 관리자가 '위치 고정'으로 명시 잠근 객체만 잠김

이유: default-locked면 플래그 없는 기존 라이브 객체가 일괄 동결되는 회귀 → 1차는 admin opt-in.

템플릿셋 탭4 도구 메뉴 노출 직접 설정 → enabledMenus로 좌측 메뉴를 한정. 빌드 플래그(requiresFlag)가 꺼진 메뉴는 화이트리스트와 무관하게 숨겨집니다.

④상품-템플릿셋 연결 방법

북모아 상품코드(sortcode)에 템플릿셋을 매핑합니다. 고객이 주문하면 연결된 템플릿셋이 자동 제공됩니다(1:N). 진입: 상품-템플릿셋 연결 관리(/product-template-sets).

데이터 모델 (ProductTemplateSet)

컬럼뜻
sortcode북모아 카테고리(상품) 코드 (예: 001001001)
prdtStanSeqno북모아 규격 번호. NULL이면 모든 규격에 적용
templateSetId연결할 템플릿셋
displayOrder표시 순서
isDefault기본 템플릿(먼저 추천)
isActive활성 여부

연결 단계

  1. 연결 추가[연결 추가] → "상품-템플릿셋 연결 추가" 모달.
  2. 상품코드(sortcode)자동완성(상품명/코드로 북모아 카테고리 검색, 2자 이상).
  3. 규격 번호(선택)비우면 해당 상품 모든 규격에 적용.
  4. 템플릿셋 선택(다중)검색·다중 선택. 셋 정보(책자/리플렛·판형)가 라벨에 표시.
  5. 기본/순서1개 선택 시 '기본 템플릿' 토글. 여러 개는 순서대로 displayOrder, 첫 번째가 기본.
  6. 추가1개=단건 생성, 여러 개=일괄 생성.

  • 순서 displayOrder 컬럼

외부 조회 (북모아 연동)

GET /api/product-template-sets/by-product?sortcode=&stanSeqno=

⚠️ API Key 인증 필요(@Public+ApiKeyGuard, X-API-Key/STORIGE_API_KEY). 관리자 CRUD는 Bearer+RolesGuard.

폴백 조회 로직

  1. sortcode+prdtStanSeqno 정확 매칭
  2. 없으면 sortcode만(규격 NULL)
  3. displayOrder 순 정렬
  4. isDefault를 첫 번째로
  5. isDeleted 템플릿셋 연결 행 제외(total도 제외 후 카운트)

북모아 edit.php → storige_get_template_sets()로 목록 수신 후 에디터 전달.
