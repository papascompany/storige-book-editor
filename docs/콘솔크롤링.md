# 디지털계산식 가격표 — 콘솔 크롤링 작업 정리

## 배경: 왜 콘솔에서 자바스크립트로 해야 했는가

기존 노리터 사이트(`bookmoa.noriter.co.kr`)는 옵션을 바꾸면 가격을 **동기 XHR**로 재계산합니다. 외부 자동화(크롬 CDP)로 시도하면:

- 동기 XHR가 진행 중일 때 페이지가 **프리즈** → CDP의 **45초 명령 한도**에 자주 걸림
- 수 분~수십 분 걸리는 추출에서 중간 끊김 다발
- `alert/confirm`이 뜨면 자동화가 멈춤

**그래서 사이트 자체의 이벤트 루프에서 돌리는 콘솔 스크립트**가 가장 안정적이라는 결론. 페이지 내부 setTimeout으로 돌리므로 시간 제한이 없고, alert도 미리 무력화합니다.

(언급하신 "포스터로 했던 작업" = **Phase H PoC(2026-05-30)**, 대상은 **디지털전단/포스터** 1종으로 스키마/스크립트/엔진을 완성했고, 결과 `docs2/digital-poc-flyer.json`이 PoC 산출물입니다.)

## 추출하는 가격 모델

라이브 사이트가 가격을 **이미 분해해서 노출**합니다(`#esti_paper`/`#esti_print`/`#esti_cut`). 역산이 필요 없습니다.

```
공급가 = 종이값 + 인쇄값 + 재단값 + 후공정판매가 = 공급가 × 1.1 (VAT)
```

|구성요소|의존 변수|추출 방식|
|---|---|---|
|**종이값**|용지(종류×평량) × 규격|장당 일정 + 소량 최소금액(floor)|
|**인쇄값**|인쇄도수 × 규격 × 수량|수량구간 카운터(대량 할인 곡선)|
|**재단값**|규격|장당 일정 + 최소금액(floor)|

대상: **상업인쇄 4종 + 디지털인쇄 9종 = 13종**(13개 상품 페이지 각각 1회 실행).

## 운영자 작업 순서 (상품 1개당)

### Step 1 — 라이브 상품 페이지 열기

예) `https://bookmoa.noriter.co.kr/p/dt_bl.html?p=goods11` — 상품마다 `goods` 번호가 다릅니다.

### Step 2 — 콘솔 열기

F12 → **Console** 탭

### Step 3 — 스크립트 실행

이 레포의 `docs2/crawl-digital-console.js`(91줄, 5KB) 전체를 복사해 콘솔에 붙여넣고 Enter.

**스크립트 동작 단계**:

1. `alert/confirm/print/open` 무력화 (프리즈 방지)
2. 수량 사다리 자동 선별: `[10, 50, 100, 200, 300, 500, 1000, 2000, 4000, 8000]` 중 상품 옵션에 있는 것만
3. **재단(cut)** 측정: 규격 스윕 (HIQ=가장 큰 수량 + 소량 10장) → 장당단가·floor
4. **인쇄(print)** 측정: 인쇄도수 × 규격 × 수량 사다리 전체 → 수량구간 곡선
5. **종이(paper)** 측정 (가장 오래 걸림): 용지 종류 × 평량 × 규격 → 결합키("아트지 백색 250g") 보관
6. 진행 로그 찍힘 (`[crawl] 종이 ...완료 (누적 read N)`)
7. 완료되면 `digital-<상품>.json` **자동 다운로드**

소요 시간: 상품당 **수 분 ~ 수십 분**(옵션 조합 수에 비례). 안정성을 위해 동기 XHR 사이 `WAIT = 1600ms` 고정 대기.

### Step 4 — JSON 적재

북모아 관리자 콘솔 → **신규가격구성** 탭 → **"크롤 JSON 적재"** 버튼으로 다운로드된 `digital-*.json` 업로드.

내부적으로 `p4-pricing-digital`(별도 DB 키, RLS: 공개 read + admin write)에 저장. 기존 `p4-pricing`(책자/낱장)과 **완전 격리**됩니다.

### Step 5 — 상품 매핑

관리자 → **상품관리** → 해당 상품 편집 → **"🧮 가격 방식"**:

- `pricingKind = 'digital'` 선택
- `digitalKey` 드롭다운에서 방금 적재한 표 선택 → 저장

(`pricingKind==='digital'`이면 ProdConfigure가 자동으로 `calcDigitalQuote`로 분기. 옵션그룹 입력은 비활성, 표에서 동적 렌더링.)

### Step 6 — 스팟 체크

공개 상품 페이지에서 옵션을 바꿔가며 견적이 라이브와 일치하는지 확인. PoC(디지털전단/포스터·아트지·A4·1000장)는 공급가 ₩505,860으로 라이브와 정확 일치 검증됨.

## 주의/한계

- 스크립트는 `dt_*` 셀렉터(`#dt_amt`/`#dt_size`/`#dt_print_tmpt`/`#dt_paper_name`/`#dt_paper`) 기반. 일부 상품 페이지가 다른 ID 패턴을 쓰면 셀렉터를 맞춰 줘야 함 → 그런 상품을 만나면 알려주시면 추출기를 보강합니다.
- `WAIT=1600ms`가 일부 느린 환경에서 부족할 수 있음 → 라인 26의 상수만 늘리면 됨.
- 추출 중 콘솔에 `[crawl] ...` 로그가 안 찍히면 정상 동작 아니므로 페이지 새로고침 후 재실행.

필요하시면 (1) 13개 상품 URL 리스트 만들기, (2) 추출기 셀렉터 다른 상품용 변형, (3) 적재된 표 검증 자동화 같은 후속 작업도 도와드릴 수 있습니다.
