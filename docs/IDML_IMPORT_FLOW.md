# InDesign(IDML) 가져오기 — 업로드·변환·등록·테스트·검증 전체 플로우

> **목적**: 인디자인 파일을 Storige 템플릿/템플릿셋으로 가져오는 **운영 전체 흐름**과 **검증 체크리스트**.
> 기술 아키텍처는 [`DESIGN_IMPORT_CONVERTER.md`](DESIGN_IMPORT_CONVERTER.md) 참조(이 문서는 "어떻게 실제로 돌리고 검증하나").
> **갱신**: 2026-06-09 · 변환모듈 인쇄 정합성 사이클(색모드/아웃라인/overprint/프레임마스킹/PNG업로드) 배포 직후.

---

## 0. 두 가지 산출물 (무엇을 만드나)

| 산출물 | 설명 | 등록 경로 |
|---|---|---|
| **① 펼침면 단품 템플릿** | 표지 펼침면 1장(날개+앞표지+책등+뒤표지+날개) 디자인을 그대로 템플릿으로. 같은 사이즈 여러 표지 디자인을 교체용으로 모음 | `POST /templates`(type=spread) |
| **② 책등 가변 템플릿셋** | 표지(펼침면) + 내지로 구성된 **책** 셋. 내지 페이지 수에 따라 **책등 폭이 런타임 자동 변동** | 방법 A: 표지 Template 등록 → 셋 폼 인계 |

> "펼침면을 이미지로 누르는" 케이스 = **하이브리드 변환 모드**(아래 §2.2): 텍스트만 편집 레이어로 살리고 나머지 디자인은 300dpi PNG 1장으로 평면화.

## 1. 전체 플로우 (7단계)

```
[1] IDML 준비   →  [2] admin 업로드  →  [3] 변환(벡터/하이브리드)  →  [4] 미리보기·경고 확인
        →  [5] 등록(단품 Template / 책등가변 셋)  →  [6] 수동 보완  →  [7] 편집기 테스트·검증
```

### [1] IDML 준비 (InDesign)
- InDesign에서 **IDML로 내보내기**(`파일 > 다른 이름으로 저장 > InDesign Markup (IDML)`). `.indd`는 직접 미지원 → `scripts/indd-to-idml.jsx`(ExtendScript)로 IDML 변환 후 입력.
- ⚠️ **폰트 미임베드**: IDML은 폰트 참조만 담음. 변환·인쇄 시 폰트가 라이브러리에 있어야 정확. (최종 인쇄는 텍스트를 **아웃라인 벡터화**하므로 폰트가 라이브러리에 로드돼 있으면 정확히 출력 — §5 참조)
- 표지 펼침면은 **날개/앞표지/책등/뒤표지/날개 5영역**이 한 스프레드에 있어야 함.

### [2] admin 업로드
- admin → **템플릿 > 템플릿관리 → "IDML 가져오기"** 버튼 → 라우트 `/templates/import`.
- 파일 드롭/선택. **포맷 자동감지**: `.idml` → 펼침면/단품, `.psd` → 단일 페이지(명함/내지 단품), `.ai` → 미지원(SVG 내보내기 우회 권장).

### [3] 변환 (벡터 vs 하이브리드)
브라우저에서 즉시 변환(jszip+fast-xml-parser). **Segmented 토글로 모드 선택**:

| 모드 | 결과 | 언제 |
|---|---|---|
| **벡터** | 도형/텍스트를 편집 가능한 객체로(무손실 벡터, svg2pdf 경로). CMYK `cmykFill` 보존 | 대부분 권장. 객체 미세조정 필요 시 |
| **하이브리드** | 텍스트만 편집 레이어 + 나머지 전부 **300dpi PNG 1장**(투명) | 복잡한 효과/그라디언트가 많아 벡터 재현이 어려울 때("펼침면을 이미지로") |

> 하이브리드/PSD 배경 PNG는 이제 **스토리지에 업로드**되어 `canvasData`엔 URL만 저장(인라인 base64 제거 → DB 비대화 방지, 2026-06-09).

### [4] 미리보기·경고 확인
- 변환 결과 **SVG 미리보기** + **검수 경고** 표시:
  - **별색(Spot)** 감지 경고 / **풀블리드 커버리지**(채움이 재단선 밖까지 닿는지) / 컴파운드패스 even-odd 등.
  - 책등 세로글씨(90°), 줄바꿈, 폴리곤 베지어 경로가 원본대로인지 눈으로 확인.

### [5] 등록 (등록 대상 라디오)
- **표지 단품**: 그대로 `POST /templates`(type=spread). 끝.
- **책등 가변 셋**(방법 A): 표지 Template 저장 후 `navigate('/template-sets/new', {state:{seedTemplateId}})` 로 **템플릿셋 폼 인계**. 폼이 표지를 셋에 자동 추가(required) + 책모드 + 판형(=표지 한 면 coverWidth/Height) 자동 설정.

### [6] 수동 보완 (자동 불가 항목)
| 항목 | 이유 |
|---|---|
| **내지 템플릿 추가** | IDML 표지에 내지가 없음. book 검증=SPREAD 정확히 1개 + PAGE ≥1개 필수 |
| **pageCountRange** | 내지 수량 범위는 상품 정책(IDML에 없음) |
| **PDF 출력 모드 / 색 처리 모드** | 단면/양면, RGB/CMYK는 상품별 설정(템플릿셋 폼 '표지·PDF 출력' 패널) |
| **에셋 구성** | 노출 라이브러리 카테고리(비우면 전역) |
| **상품 연결** | `ProductTemplateSet`(sortcode) — 이 흐름 밖 별도 단계 |

### [7] 편집기 테스트·검증
- 등록된 셋으로 **편집 세션** 진입(고객 주문 또는 테스트 세션). 책모드면 스프레드 편집.
- 책등 가변·색·폰트·인쇄 출력 검증(§4, §5).

## 2. 시나리오별 상세

### 2.1 시나리오 A — 펼침면 단품 템플릿 (하이브리드 "이미지로 누르기")
1. IDML 업로드 → **하이브리드** 선택 → 텍스트만 편집 레이어 + 배경 PNG.
2. 미리보기에서 텍스트 위치/폰트 + 배경 이미지 확인.
3. 등록 대상 = **표지 단품** → `POST /templates`.
4. (옵션) 같은 사이즈의 다른 표지 디자인들을 추가로 변환·등록 → 편집기 '템플릿' 메뉴에서 **교체 가능**(③ 커버 교체 워크플로우).
5. 검증: 편집기에서 텍스트 미세조정(폰트/크기/효과) → 저장 → 인쇄 PDF 출력 확인.

### 2.2 시나리오 B — 책등 가변 템플릿셋
1. IDML 업로드 → 벡터(또는 하이브리드) → 등록 대상 = **책등 가변 셋**.
2. 표지 Template 저장 → 셋 폼 자동 인계(표지 seed + 책모드 + 판형).
3. **수동**: 내지 템플릿 추가 + `pageCountRange`(예: `[10,20,30,40]`) + PDF/색 모드 설정.
4. 셋 저장 → 상품 연결(ProductTemplateSet).
5. **책등 가변 검증**(핵심): 편집 세션에서 **내지 페이지 수 변경** → 책등 폭이 `(pageCount/2)×thickness + margin` 으로 자동 재계산 + 앞/뒤표지가 책등 변화만큼 평행이동. 편집완료→재로드 후에도 유지(`meta` 직렬화 보존).

## 3. 책등 가변 원리 (왜 자동인가)
- 책등 폭은 **템플릿 고정값이 아니라 런타임 파생값**: `spine.service.ts` `spineWidth=(pageCount/2)×thickness+margin`.
- 권위 검증 `validateSpreadAgainstAuthority`(types)는 **coverWidthMm/coverHeightMm/wingEnabled/wingWidthMm 4필드만** 비교(spineWidthMm/totalWidthMm는 의도적 제외).
- → IDML 원본의 고정 책등은 버리고 **표지 한 면(cover)+날개(wing)만 권위로 추출**하면 자연스럽게 가변화. 책등 텍스트는 중앙 재배치, 앞/뒤표지는 평행이동(리플로우 손실 없음).

## 4. 검증 체크리스트

### 4.1 변환 충실도 (admin 미리보기 + 편집기)
- [ ] z-순서(쌓임) 원본대로(원형이 책등 패널 위로 안 올라오는지 등)
- [ ] 책등 제목/저자 **세로글씨(90°)** 정상
- [ ] 뒷면 출판사 등 **여러 줄 텍스트** 줄바꿈 보존(영역 이탈 없음)
- [ ] 폴리곤/곡선 **베지어 경로** 복원
- [ ] mm 단위 치수 정확(총폭 = 날개+앞+책등+뒤+날개)

### 4.2 인쇄 규약 (이번 사이클 신규 — §5)
- [ ] **색 처리**: 셋 `colorMode` 설정대로(RGB 유지 / CMYK 변환). ※ 워커 색변환은 적용 후 검증
- [ ] **텍스트**: 인쇄 PDF에서 텍스트가 **아웃라인 벡터**(폰트 의존 없음), 혼합폰트 각 run 제 폰트
- [ ] **overprint/별색**: 고객 PDF의 `/OP`·Separation이 출력에 보존
- [ ] **프레임**: 사진을 프레임에 채우면 **투명창에만** 보이고 밖은 가려짐

### 4.3 책등 가변 / 워크플로우
- [ ] 내지 수 변경 → 책등 폭 자동 재계산 + 앞/뒤표지 평행이동
- [ ] 편집완료→재로드 후 책등 가변 재배치 유지(`meta` 보존)
- [ ] 같은 사이즈 다른 표지 펼침면으로 **교체**(편집기 '템플릿' 메뉴)

|---|---|---|

| **텍스트 아웃라인(혼합폰트 충실도)** | 텍스트를 벡터 아웃라인으로 출력(폰트 미임베드 무관), 혼합폰트 각 폰트 정확 | ✅ 배포(merge 20ac84c) |
| **overprint/별색 보존** | 고객 PDF의 overprint·별색이 재distill 시 소실되지 않음 | ✅ 배포(워커) |
| **사진 프레임 마스킹** | 프레임 투명창에만 사진(inverted clipPath) | ✅ 배포 |

> ⚠️ 위 인쇄출력 기능은 **실주문/시각 검증 중**(deploy-and-iterate). 문제 발견 시 §6.

## 5b. 재편집 정합성 사이클 (2026-06-11, 등록→재편집→저장 무결성)

가져온 템플릿을 **다시 '편집'으로 열었을 때** 객체가 투명/붕괴되거나, 미리보기·하이브리드 래스터가
좌우로 어긋나거나, 배경 아트워크가 이동되던 5종 결함을 수정(서브에이전트 교차검증, commit `527b85b`).

| # | 증상 | 근본원인 | 수정 |
|---|---|---|---|
| ① | 재편집 시 객체가 투명(선택 프레임만)·일부만 보임 | `loadJSON` 이 `canvasData.width/height`(판형 메타)를 fabric 캔버스 px 치수로 덮어씀 → `skipOffscreen` 컬링이 화면 밖 객체 렌더 스킵. IDML 템플릿은 workspace 객체가 없어 `afterLoad` 의 치수 복구도 누락 | `ServicePlugin.loadJSON` width/height strip + `WorkspacePlugin.afterLoad` workspace 복원 + `SpreadPlugin.afterLoad` 가이드 재렌더 |
| ② | 저장된 좌표가 영구 변위(특정 객체만) | ①로 깨진 화면에서 **사용자가 객체를 드래그**한 산물이 DB 박제(코드 버그 아님) | ① 수정으로 화면 정상화 → 재발 방지. 이미 변위된 템플릿은 **재가져오기** 필요 |
| ③ | admin 임포트 미리보기에서 앞표지 패널이 통째로 안 보임 | `preview/svg.mjs` 가 중앙원점 객체를 좌상단 viewBox 에 보정 없이 그림(a92f146 누락) | 비-path 객체에 `+halfW/+halfH` 보정 |
| ④ | 하이브리드 PNG 표지 좌우가 뒤바뀜 | `raster/rasterize.mjs` 동일 누락 | 동일 보정 |
| ⑤ | 배경 아트워크가 이동·삭제됨 | 변환기가 `selectable:true` 로 emit | `ARTWORK_LOCK`(고정·패널숨김·관리자잠금) emit + `useTemplateSetSave` 직렬화 전체 보존 |

> **배포 측 함정 동반 수정**: `apps/admin/vercel.json` 의 `ignoreCommand` 에 `packages/indesign-import` 가
> 빠져 있어 **변환기만 고치면 admin 이 재배포되지 않던** 문제 해소(등록 하이브리드 `styles` 누락의 배포측 원인).
> 변환기 수정 후 admin 배포가 안 보이면 이 항목 확인.

> ⚠️ **이미 등록된 깨진 템플릿**(2026-06-11 이전 변환·또는 ②로 변위 저장)은 코드 수정으로 자동 복원되지 않음 →
> **재가져오기**(또는 DB 좌표 패치) 필요. 신규 가져오기는 정상.

## 6. 문제 해결 (자주 생기는 이슈)

| 증상 | 점검 |
|---|---|
| 변환 텍스트 폰트가 다름/깨짐 | 해당 폰트가 라이브러리에 시딩됐는지. 미임베드 폰트는 폴백 → 아웃라인 시 base 폰트(또는 래스터) |
| **재편집 시 객체가 투명/사라짐** | §5b ① — 2026-06-11(`527b85b`) 이후 신규 가져오기는 정상. 그 이전 등록본은 **재가져오기**. 배포 반영은 새 탭/캐시버스트로 확인 |
| **미리보기/하이브리드 PNG 좌우 어긋남** | §5b ③④ — `527b85b` 이후 정상. admin 배포 반영 확인(`ignoreCommand` 수정 포함) |
| **배경 아트워크가 움직임/지워짐** | §5b ⑤ — 신규 가져오기는 `template-element` 로 고정. 교체는 재가져오기 |
| 객체가 영역 밖으로 이탈 | 미리보기에서 확인 → 편집기 템플릿편집 모드로 미세조정(배경 아트워크는 고정, 텍스트만 편집) |
| 책등이 안 변함 | 책모드 셋인지, SPREAD 1개+PAGE≥1개인지, `meta`(regionRef) 보존됐는지 |
| 책 셋 저장 안 됨 | book 검증=SPREAD 정확히 1개 필수 → 내지(PAGE) 추가했는지 |
| 표지 색이 화면과 다름 | RGB 파이프라인 색차. 셋 `colorMode=cmyk` + 워커 색변환(적용 후) |
| 프레임에 사진이 삐져나옴/반대로 가려짐 | 프레임 마스킹 방향(inverted) 검증 항목 — 반대면 보고 |
| 변환 PNG가 안 보임 | 스토리지 업로드 URL(`/storage/...`) 접근성(CORS) 확인 |
