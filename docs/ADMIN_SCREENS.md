```
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**항목**:
- **현재 비밀번호** (`currentPassword`, 필수) — 본인 확인용. 서버에서 `bcrypt.compare`로 검증, 불일치 시 거부.
- **새 비밀번호** (`newPassword`, 필수) — 최소 8자 검증(클라이언트 + 서버).
- **새 비밀번호 확인** (`confirmPassword`, 필수) — 새 비밀번호와 일치 검증(클라이언트). 불일치 시 인라인 오류.

**동작**:
- `[비밀번호 변경]` → `PATCH /api/auth/change-password` 호출(`{ currentPassword, newPassword }`).
- 성공 시: 성공 메시지(토스트) 노출 + 세 입력 필드 초기화(폼 리셋).
- 실패 시: 서버 오류(현재 비밀번호 불일치·검증 실패)를 필드/토스트로 표시.

**권한**:
- 전역 `JwtAuthGuard` 보호 라우트. **로그인한 관리자 본인만** 자기 계정의 비밀번호 변경(서버는 `@CurrentUser().id` 기준 본인 레코드만 갱신).
- 비밀번호 변경 엔드포인트는 rate limit(`@Throttle`) 적용.

> ℹ️ 평문 비밀번호는 화면·로그·문서 어디에도 노출하지 않음(입력은 마스킹, 서버는 `bcrypt` 해시 저장).
