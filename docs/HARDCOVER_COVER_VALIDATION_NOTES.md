# 하드커버 표지 사이즈·검증 기준 메모 (오너 결정 2026-07-13)

> 출처: 가로형 템플릿 6이슈 트랙(fix/landscape-template-ux-20260713) 중 하드커버 가로
> templateSet(83e6ec80) 판형 교정 결정 시 오너가 정리한 표지 규격 원칙.
> 이 문서는 **결정 기록 + 후속 트랙 스코프**다. 즉시 코드 변경은 판형 데이터 교정뿐.

## 1. 확정 원칙

1. **판형(templateSet.width/height) = 내지 재단 사이즈.** 내지 PDF 검증(T3: 재단/재단+2×bleed 비교,
   도련 자동삽입)의 기준값이다. 하드커버 가로 A4 책자 = 297×210 (bleed 3 → 작업 303×216).
   - 83e6ec80(A4하드커버 가로)의 301×214는 이 규약 위반으로 **297×210으로 교정**(2026-07-13).
2. **표지는 판형에 종속되지 않는다.** 인쇄형 하드커버 표지는 관리자가 펼침면(스프레드)으로
   제작·등록한 사이즈(= 앞표지+책등+뒤표지+**싸바리**(보드를 감싸 표지 뒷면에 풀로 접어
   넘기는 마감 여분) 포함)가 정본이며, **표지 PDF는 그 제작 사이즈와 입력값으로 생성**되어야
   한다. 가로형/세로형 책 방향에 따라 표지 크기도 당연히 달라진다.
3. **표지 파일 검증도 같은 기준**: 표지 산출물/업로드 검증은 판형이 아니라
   "관리자 제작 표지 사이즈(책등+싸바리 포함)"와 비교해야 한다.
4. **책등 가변**: 페이지 수에 따라 책등 폭이 변하는 상품은 기대 표지 사이즈를
   **계산해서** 적용해야 한다(고정 상수 비교 금지). 편집기에는 이미 spine 재계산
   (debouncedRecalcSpine / SpreadPlugin resizeSpine) 로직이 존재 — 검증 측 기준값
   산출도 동일 산식을 공유해야 한다.

## 2. 커버 유형 분류 (오너 정리)

| 유형 | 특성 | 검증/산출 기준 |
|---|---|---|
| 인쇄 커버 (펼침면 표지 템플릿) | 관리자가 스프레드로 제작·등록 | 제작 사이즈(책등+싸바리 포함) 그대로 PDF 생성·검증. 책등 가변 시 계산 |
| 페브릭/래더 커버 | 비인쇄 커버 (화보집·압축앨범 등) | **후속 실무 검증 필요** — 표지 인쇄 PDF 부재/별도 산출물 여부 포함 |
| 누드제본 커버 | 조건이 상이 | **후속 실무 검증 필요** |

## 3. 후속 트랙 (이번 배포 범위 밖 — 오너 지시)

- 페브릭/래더 커버와 누드제본 커버의 **표지 검증 규칙은 실무 검증 후 별도 수정 작업**으로
  진행한다(오너와 함께 실물 기준 확정 → 검증 파이프라인 반영).
- 인쇄형 표지의 업로드/산출 검증이 현재 코드에서 "제작 사이즈+가변 책등 계산" 기준을
  온전히 따르는지 전수 확인도 이 트랙에서 함께 본다(이번 트랙은 내지(T3)만 다뤘다).
