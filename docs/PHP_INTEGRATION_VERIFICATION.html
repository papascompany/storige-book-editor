Storige 에디터 — PHP 연동 검증 가이드

북모아(PHP) 측에서 Storige 에디터를 연동할 때 구현·검증해야 할 항목을 정리한 문서입니다.
Storige 측(API·Worker·에디터)은 이미 운영 배포 완료 + Sentry 운영 추적 활성화 상태입니다.

대시보드: sentry.io/organizations/papascompany/issues/
유용한 endpoint: GET /api/health (인증 불필요, 큐 상태 포함) — PHP cron으로 5분마다 호출해 가용성 모니터링 가능
자세한 내용은 시스템 통합 문서 §운영 모니터링 참조.
⚠️ 2026-05-03 사용자 격리 보안 패치 — PHP 측 영향
PHP 측 기존 코드는 그대로 작동하지만 다음 사항을 반드시 알고 진행해야 합니다:
  • 🔴 /files/:id/download 인증 강제 — 합성 결과 PDF 다운로드는 /files/:id/download/external (X-API-Key) 사용
  • 🔴 fileId / sessionId 클라이언트 노출 금지 — 모든 storige API 호출은 PHP 서버에서
  • 🔴 /edit-sessions/:id, /files/:id 소유자 검증 — JWT.memberSeqno 일치 시만 접근
  • 🟡 shop-session 호출 시 orderSeqno 추가 권장 — JWT에 포함되어 자동 검증 (호환 유지)
  • 🟡 callbackUrl 호스트 화이트리스트: papascompany.co.kr, bookmoa.com만 자동 허용

1 에디터 URL 파라미터 수정

⚠️ Breaking Change — 기존 파라미터 pages, wingFront, wingBack은 더 이상 작동하지 않습니다. 아래 표를 참고해 코드를 수정하세요.

파라미터 변경 요약

PHP URL 생성 함수 (권장 방식)

PHP
<?php
// ✅ Storige 에디터 URL 생성 함수
function buildStorigeEditorUrl(array $params): string {
    $base = 'https://editor.papascompany.co.kr';

    // ─── 필수 파라미터 ───
    $query = [
        'template_set_id' => $params['templateSetId'],
        'api_base_url'    => 'https://api.papascompany.co.kr/api',
    ];

    // ─── 주문 번호 (권장) ───
    if (!empty($params['orderSeqno'])) {
        $query['order_seqno'] = $params['orderSeqno'];
    }

    // ─── 페이지 수 (⚠️ 'pages' 아님, 'pageCount' 사용) ───
    if (!empty($params['pageCount'])) {
        $query['page_count'] = $params['pageCount'];
    }

    // ─── 종이·제본 (책등 폭 자동 계산) ───
    if (!empty($params['paperType'])) {
        $query['paper_type']   = $params['paperType'];    // 예: 'mojo_80g'
        $query['binding_type'] = $params['bindingType'];  // 예: 'perfect'
    }

    // ─── 완료 후 이동 URL ───
    if (!empty($params['returnUrl'])) {
        $query['return_url'] = $params['returnUrl'];
    }

    // ─── 재편집: 기존 세션 복원 ───
    if (!empty($params['sessionId'])) {
        $query['session_id'] = $params['sessionId'];
    }

    // ─── 옵션 C: 자유 사이즈 (Admin에서 허용한 상품만) ───
    if (!empty($params['widthMm']) && !empty($params['heightMm'])) {
        $query['width']  = $params['widthMm'];
        $query['height'] = $params['heightMm'];
    }

    return $base . '?' . http_build_query($query);
}

// ─── 사용 예시 ───
$url = buildStorigeEditorUrl([
    'templateSetId' => 'ts-perfect-a5-50p',
    'orderSeqno'    => $order['seqno'],
    'pageCount'     => 50,
    'paperType'     => 'mojo_80g',
    'bindingType'   => 'perfect',
    'returnUrl'     => '/mypage/order_edit.php?order_seqno=' . $order['seqno'],
]);
💡 에디터 진입 방식 — Storige 에디터는 iframe이 아닌 직접 URL 이동 방식입니다. 사용자를 에디터 URL로 보내면 편집이 끝난 후 returnUrl로 돌아옵니다.

4 Worker API 인증 & PDF 합성 요청

⚠️ 인증 방식 변경
기존: Authorization: Bearer {JWT 토큰}
현재: X-API-Key: {발급받은 API 키}
API Key는 Storige 인프라팀에서 발급. 환경별(개발/운영)로 별도 키 사용 권장.

Step 1 — 병합 가능 여부 사전 확인

주문 직전에 PDF 병합이 가능한지 먼저 확인합니다. 페이지 수 불일치 등의 오류를 미리 잡을 수 있습니다.

PHP
function checkMergeable(string $sessionId, string $coverFileId, string $contentFileId, float $spineWidth): array {
    $payload = [
        'editSessionId' => $sessionId,
        'coverFileId'   => $coverFileId,
        'contentFileId' => $contentFileId,
        'spineWidth'    => $spineWidth,   // mm 단위
    ];

    $ch = curl_init('https://api.papascompany.co.kr/api/worker-jobs/check-mergeable/external');
    curl_setopt_array($ch, [
        CURLOPT_POST           => true,
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_HTTPHEADER     => [
            'X-API-Key: ' . STORIGE_API_KEY,   // ⚠️ JWT 아님!
            'Content-Type: application/json',
        ],
        CURLOPT_POSTFIELDS     => json_encode($payload),
        CURLOPT_TIMEOUT        => 30,
    ]);

    $resp = json_decode(curl_exec($ch), true);
    curl_close($ch);

    // 응답: { mergeable: true/false, issues: [{ code, message }] }
    return $resp;
}

// ─── 사용 예시 ───
$check = checkMergeable($sessionId, $coverFileId, $contentFileId, $spineWidthMm);
if (!$check['mergeable']) {
    $errorMsg = implode(', ', array_column($check['issues'], 'message'));
    throw new Exception('PDF 병합 불가: ' . $errorMsg);
}

function requestSynthesis(array $params): array {
    $payload = [
        'editSessionId' => $params['sessionId'],
        'coverFileId'   => $params['coverFileId'],
        'contentFileId' => $params['contentFileId'],
        'spineWidth'    => $params['spineWidthMm'],

        'priority'      => 'high',   // 'high' | 'normal' | 'low'

        CURLOPT_POST           => true,
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_HTTPHEADER     => [
            'X-API-Key: ' . STORIGE_API_KEY,
            'Content-Type: application/json',
        ],
        CURLOPT_POSTFIELDS     => json_encode($payload),
        CURLOPT_TIMEOUT        => 30,
    ]);

    $resp = json_decode(curl_exec($ch), true);
    curl_close($ch);

    // 응답: { jobId: "uuid", status: "PENDING", createdAt: "..." }

5 Webhook 수신 구현 — 신규 파일 작성 필요

📂 파일 위치 — /storige/proc/synthesis_callback.php 파일을 새로 만들어야 합니다.
Storige Worker가 PDF 합성을 완료하거나 실패하면 이 URL로 POST 요청을 보냅니다.
PHP — /storige/proc/synthesis_callback.php (신규 작성)
<?php
// Storige Worker가 PDF 합성 완료/실패 시 POST로 호출하는 파일

$rawBody = file_get_contents('php://input');
$payload = json_decode($rawBody, true);
$event   = $_SERVER['HTTP_X_STORIGE_EVENT'] ?? $payload['event'] ?? '';

// 디버깅용 로그 (나중에 제거 가능)
error_log('[storige_webhook] event=' . $event . ' jobId=' . ($payload['jobId'] ?? 'n/a'));

if ($event === 'synthesis.completed') {
    /*
     * 받는 데이터:
     * {
     *   "event":         "synthesis.completed",
     *   "jobId":         "uuid",
     *   "orderId":       "ORD-12345",
     *   "status":        "COMPLETED",
     *   "outputFileUrl": "https://api.papascompany.co.kr/api/files/.../download",
     *   "completedAt":   "2026-05-02T10:00:00Z"
     * }
     */
    $jobId         = $payload['jobId'];
    $orderId       = $payload['orderId'];           // 'ORD-12345' 형식
    $outputFileUrl = $payload['outputFileUrl'];     // 최종 병합 PDF URL ✅

    // 'ORD-12345' → 12345 파싱
    $orderSeqno = (int)str_replace('ORD-', '', $orderId);

    // DB 업데이트: PDF URL 저장 + 상태 변경
    updateOrderFile($orderSeqno, $outputFileUrl);
    updateOrderStatus($orderSeqno, 210);  // 210 = PDF 생성 완료

} elseif ($event === 'synthesis.failed') {
    $orderId      = $payload['orderId'];
    $errorMessage = $payload['errorMessage'] ?? '알 수 없는 오류';
    $orderSeqno   = (int)str_replace('ORD-', '', $orderId);

    updateOrderStatus($orderSeqno, -1);   // 실패 상태
    notifyAdminOfSynthesisFailure($orderSeqno, $errorMessage);
}

// ⚠️ 반드시 200 응답해야 함 (미응답 시 Storige가 최대 3회 재시도)
http_response_code(200);
echo json_encode(['received' => true]);
exit;

? FAQ

API Key는 어떻게 발급받나요? ▶
Storige 인프라팀에 요청하면 발급됩니다. 개발 환경과 운영 환경에 각각 다른 키를 사용하는 것을 권장합니다. 이 문서에는 키 값을 기재하지 않으니 별도 채널로 전달받으세요.
sessionId 없이 orderSeqno만 전달하면 어떻게 되나요? ▶
Storige API가 해당 orderSeqno로 기존 세션을 자동 검색합니다. 기존 세션이 있으면 불러오고, 없으면 새 세션을 생성합니다. 즉, 재편집은 자동으로 처리됩니다.
Webhook 없이 운영하고 싶다면? ▶
GET /api/worker-jobs/external/{jobId}를 주기적으로 호출해 상태를 확인하는 폴링 방식을 사용할 수 있습니다. 단 실시간성이 떨어지고 서버 부하가 있을 수 있습니다.
PDF 합성 없이 에디터만 사용하고 싶다면? ▶
synthesize/external 호출 없이, 편집 완료 후 받은 coverFileId/contentFileId UUID를 Storige API GET /files/{fileId}를 통해 직접 파일 URL로 변환해 다운로드할 수 있습니다.
pages 파라미터가 왜 pageCount로 바뀌었나요? ▶
기존 pages는 표지 포함 전체 페이지 수였습니다. Storige의 내부 모델은 내지(본문) 페이지 수를 기준으로 표지를 별도 계산하는 구조라 혼선 방지를 위해 명칭을 변경했습니다.
Webhook 수신 파일(synthesis_callback.php)에 보안 검증이 필요한가요? ▶
현재는 X-Storige-Signature 헤더 기반 HMAC 서명 검증을 코드에 주석으로 포함해 두었습니다. Storige 인프라팀과 웹훅 시크릿 키를 합의한 후 주석을 해제해 활성화하세요. 당장 운영에서는 IP 화이트리스트로 대체 가능합니다.

11-1. 합성 결과 PDF 다운로드 — /external endpoint 사용

기존 /files/:id/download는 인증 강제 — PHP 서버에서 합성 결과 PDF 가져갈 때는 신규 endpoint 사용:

// ✅ PHP 서버에서 합성 결과 다운로드 (X-API-Key 인증)
function downloadSynthesizedPdf(string $fileId): string {
  $ch = curl_init("https://api.papascompany.co.kr/api/files/{$fileId}/download/external");
  curl_setopt_array($ch, [
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_HTTPHEADER => ['X-API-Key: ' . STORIGE_API_KEY],
  ]);
  $pdfData = curl_exec($ch); // 200 OK 시 PDF 바이너리
  curl_close($ch);
  return $pdfData;
}
⚠️ 브라우저(클라이언트)에서 직접 다운로드 시도 → 401. 사용자에게 보여줄 때 PHP 서버에서 받아 다시 스트림으로 전달.

function getMyFiles(int $memberSeqno): array {
  $ch = curl_init("https://api.papascompany.co.kr/api/files?memberSeqno={$memberSeqno}");
  curl_setopt_array($ch, [
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_HTTPHEADER => ['X-API-Key: ' . STORIGE_API_KEY],
  ]);
  $res = json_decode(curl_exec($ch), true);
  return $res['files'] ?? [];
}

11-5. Webhook callbackUrl 호스트 검증

PHP 측 webhook 수신 URL은 다음 호스트여야 함 (자동 허용):

  • *.papascompany.co.kr (서브도메인 포함)
  • *.bookmoa.com
  • localhost, 127.0.0.1 (개발 환경)

다른 호스트로 webhook 호출 시 storige API에서 거부 + 로그 기록. 운영 시 추가 호스트 필요하면 VPS ~/storige/.env에 WEBHOOK_ALLOWED_HOSTS=papascompany.co.kr,bookmoa.com,my-test.example.com 형식으로 추가.
