# ① 템플릿 등록 방법

템플릿을 만드는 길은 **3가지**입니다.

| 방법 | 진입 | 소스 | 산출물 |
|---|---|---|---|
| **1-A. IDML 변환** | 템플릿 관리 → `IDML 가져오기` | InDesign `.idml`(표지 펼침면) | `type=spread` 템플릿 |
| **1-B. PSD 변환** | 템플릿 관리 → `IDML 가져오기`(같은 화면, 자동감지) | Photoshop `.psd`(단품) | `type=page` 또는 `cover` 템플릿 |
| **1-C. 직접 수기 등록** | 템플릿 관리 → `템플릿 생성` | 빈 캔버스에서 직접 디자인 | 모든 타입 |

> 진입: admin 좌측 메뉴 **템플릿 > 템플릿 관리**(`/templates`). 우상단에 **[IDML 가져오기]**, **[템플릿 생성]** 두 버튼.

## 1-A. IDML 변환 등록 (케이스별)

InDesign에서 만든 **표지 펼침면**(날개+앞표지+책등+뒤표지+날개 5영역)을 템플릿으로 변환합니다.

### 사전 준비 (InDesign)
1. InDesign에서 **IDML로 내보내기**: `파일 > 다른 이름으로 저장 > InDesign Markup (IDML)`.
   - `.indd`는 직접 미지원 → `scripts/indd-to-idml.jsx`(ExtendScript)로 변환 후 사용.
2. ⚠️ **폰트는 IDML에 임베드되지 않습니다**(참조만 담김). 변환 화면의 **폰트 매칭**에서 누락 폰트를 시딩해야 정확히 출력됩니다.
3. ⚠️ **배치(placed) 이미지는 IDML에 픽셀이 없습니다**(링크 메타만). 원본 이미지를 같은 드롭에 함께 올리면 자동 복원됩니다.

### 등록 단계 (`/templates/import`)

```
[1] 변환 방식 선택  →  [2] 파일 업로드  →  [3] 미리보기·매칭·경고 검수
        →  [4] 등록 대상 선택  →  [5] 저장
```

**[1] 변환 방식 선택 — 케이스별 3종 (가장 중요)**

화면 상단 `변환 방식 (IDML)` Segmented 토글에서 선택합니다. 선택값은 `spreadConfig.conversionMode`로 저장됩니다.

| admin 문구 (mode) | conversionMode | 동작 | 언제 쓰나 |
|---|---|---|---|
| **벡터 (전체 편집형)** `vector` | `full` | 도형·텍스트를 **모두 편집 가능한 벡터**로 추출(정밀, CMYK `cmykFill` 보존) | **대부분 권장**. 단순한 디자인, 객체 미세조정 필요 시 |
| **펼침면 플랫형 (책등 고정)** `hybrid` | `flat-spread` | 텍스트만 편집 레이어 + 나머지 디자인을 **전폭 300dpi PNG 1장**으로 평면화 | 복잡한 효과·그라디언트가 많아 벡터 재현이 어려울 때 + **책등 두께 고정 상품**(페이지 수 변동 없음). ⚠️ 편집기에서 책등 가변 차단 |
| **책등가변 3분할 플랫형** `flat-spine` | `flat-spine` | 텍스트만 편집 레이어 + 디자인을 **뒷표지/책등/앞표지 3장의 300dpi PNG**로 분할 고정. 책등 PNG는 책등 중심 기준 **3배폭** → 페이지 수에 따라 책등 두께가 변해도 디자인이 따라감 | 복잡한 디자인을 **책등 가변 책 셋**으로 쓸 때(권장) |

> **변환 방식 결정 트리**
> ```
> 디자인이 단순한가? ──── 예 ──→ [벡터 (전체 편집형)]   ← 기본값·권장
>        │
>        아니오(복잡한 효과/그라디언트)
>        │
>        ▼
> 책등 두께가 페이지 수에 따라 변하나?
>        │
>   ┌────┴────┐
>  아니오     예
>   │         │
>   ▼         ▼
> [펼침면     [책등가변
>  플랫형]     3분할 플랫형]
> ```

**[2] 파일 업로드**

드롭존(`IDML·패키지 zip·PSD (+배치 이미지)를 끌어다 놓거나 클릭`)에 업로드. 허용 형식: `.idml .psd .zip .jpg .jpeg .png .gif .webp .bmp .avif`

배치 이미지 복원 케이스 4가지:
1. **`.idml` 단독** — 기존 동작(회색 플레이스홀더 + 경고).
2. **`.idml` + 이미지 다중 선택** — 같은 드롭에 JPG/PNG 등을 함께 → **파일명으로 자동 매칭**(NFC 정규화, 대소문자 무시, 프레임 크롭/플립까지 베이크).
3. **패키지 zip** — `*.idml` + 이미지를 zip 1개로(InDesign 패키지 Links 폴더 구조 무관, 파일명 기준).
4. **이미지 추가 업로드** — 변환 후 매칭 실패 목록을 보고 이미지를 추가로 올리면 **직전 변환에 병합 재변환**.

> ⚠️ TIFF/EPS/PDF/AI 등 브라우저 미지원 형식은 건너뜀 → **JPG/PNG로 변환해 재업로드**. AI(.ai)는 미지원(InDesign에 배치 후 IDML 또는 SVG 내보내기 우회).

**[3] 미리보기·매칭·경고 검수**

- **SVG 미리보기**: z-순서, 책등 세로글씨(90°), 줄바꿈, 베지어 경로가 원본대로인지 육안 확인.
- **감지된 사양**: 표지 한 면(coverWidth) · 표지 높이 · 책등(spineWidth, **가변**) · 날개 · 재단여백 · **총폭**.
- **배치 이미지 매칭**(동반 업로드 시): `✓ 복원` / `참조 없음` / `✗ 실패`(사유 표시).
- **폰트 매칭 (라이브러리)**: 변환 폰트별로 `✓ 사용 가능` / `비활성`(→[활성화]) / `✗ 미등록`(→[폰트 파일 등록] TTF/OTF/WOFF2). 미등록 폰트 텍스트는 편집기에서 기본 폰트로 대체됩니다.
- **검수 경고**(`검수 필요 — 변환 손실/주의 항목`): 별색(Spot) · 풀블리드 커버리지 · 자간/단락정렬 혼합(지배값 근사) · 장평(HorizontalScale 미적용) · 미해석 색상(검정 대체) · 텍스트 fill 그라디언트(검정 대체 보류) · 세로쓰기 등.

**[4] 등록 대상 선택** (IDML 전용 라디오)

| 선택 | 동작 |
|---|---|
| **표지 템플릿만 등록 (펼침면 단품)** | `type=spread` 템플릿만 생성 → 저장 후 편집기로 이동. 셋 구성은 추후 |
| **책등 가변 셋으로 이어서 등록 (표지 + 내지 책)** | 표지 템플릿 생성 후 **템플릿셋 폼으로 자동 인계**(`/template-sets/new`, `seedTemplateId` 전달). 표지가 셋에 자동 추가 + 책모드 + 판형(표지 한 면 크기) 자동 설정 |

> ⚠️ **'펼침면 플랫형(책등 고정)'으로 등록한 표지를 책등 가변 셋에 넣으면 책등이 변하지 않습니다**(의도된 가드 — 통짜 PNG 보호). 책등 가변이 필요하면 **'책등가변 3분할 플랫형'**으로 재변환하세요.

**[5] 저장**

- 템플릿명 입력(기본값: `<파일명> (가져옴)`), 카테고리 선택(선택).
- 버튼: **[템플릿으로 저장]**(단품) 또는 **[저장 후 셋 등록으로 →]**(책등 가변 셋).
- 저장 시 인라인 base64 PNG 배경은 자동으로 스토리지에 업로드되어 URL로 치환됩니다(DB 비대화 방지).

## 1-B. PSD 변환 등록

Photoshop **단품 디자인**(명함·내지 단품 등)을 단일 페이지 템플릿으로 변환합니다. **IDML과 같은 화면**(`/templates/import`)에서 `.psd` 업로드 시 **자동 감지**됩니다.

### IDML과의 차이

| 항목 | IDML | PSD |
|---|---|---|
| 산출물 | 펼침면(spread) | **단일 페이지**(page/cover) |
| `spreadConfig` | 있음(5영역·책등 가변) | **없음** |
| 변환 방식 토글 | 3종 | 없음(텍스트 레이어 + 배경 합성 PNG 고정) |
| 배치 이미지 동반 업로드 | 사용 | **미사용**(픽셀이 PSD에 내장) |

### 등록 단계
1. `.psd` 파일 업로드 → 자동으로 PSD 모드.
2. 미리보기 + 감지된 사양(판형 가로×세로 · 편집 텍스트 레이어 수 · 배경 합성 레이어 → 배경 PNG 1장).
3. **페이지 타입** 라디오 선택:
   - **내지/단품 (page)** → `type=page`
   - **표지 (cover)** → `type=cover`
4. 템플릿명·카테고리 입력 → **[템플릿으로 저장]**.
5. 저장 후 편집기로 이동 → 텍스트 폰트·크기·효과를 확정.

> 텍스트는 편집 레이어로 살아있고, 나머지(사진·효과)는 배경 PNG 1장으로 합성 고정됩니다.

## 1-C. 직접 수기 등록 (디자인 요소 수기 배치)

빈 캔버스에서 직접 객체를 배치해 템플릿을 만듭니다. 변환 소스가 없거나, 단순한 양식/배경/도형 위주 템플릿에 적합합니다.

### 등록 단계 (`/templates/editor`)
1. 템플릿 관리 → **[템플릿 생성]** 클릭 → **"새 템플릿 설정"** 모달.
2. **템플릿 이름** 입력.
3. **템플릿 타입** 선택:
   | 타입 | 용도 |
   |---|---|
   | 내지 (Page) | 책 내지 / 단품 페이지 |
   | 표지 (Cover) | 단면 표지 |
   | 책등 (Spine) | 책등 |
   | 날개 (Wing) | 표지 날개 |
   | 스프레드 (Spread) | 책모드용 표지 펼침면 |
   | 면지 (EndPaper) | 표지 안쪽 / 뒤표지 안쪽 |
4. **치수 입력**:
   - 일반 타입: **가로/세로 (mm)** (10~1000).
   - 스프레드: **"스프레드 템플릿 설정"** 모달에서 표지 한 면(coverWidthMm/coverHeightMm) · 날개(wingEnabled/wingWidthMm) · 초기 책등(입력 필드명 `initialSpineWidthMm`, 셋 저장 키는 `spineWidthMm`) 입력(책등은 셋에서 가변).
5. **편집기 진입**: 설정대로 편집기(iframe)가 열립니다 → 텍스트/이미지/요소(SVG·도형)/배경/프레임/QR 등을 직접 배치.
6. **저장**: 편집기 저장 → admin이 `TEMPLATE_SAVED` 수신 → "템플릿이 저장되었습니다."

> 수기 등록한 템플릿도 IDML/PSD 변환 결과와 동일하게 템플릿셋에 넣고 상품에 연결할 수 있습니다.

# ② 템플릿셋 등록 방법

여러 템플릿(표지+내지+면지)을 한 상품의 편집 단위로 묶고, 출력·면지·에셋·도구 메뉴를 설정합니다.

### 진입 2가지
| 경로 | 설명 |
|---|---|
| **방법 A — IDML 인계** | 1-A의 등록 대상에서 '책등 가변 셋으로 이어서 등록' → 표지가 미리 추가된 폼으로 자동 진입 |
| **방법 B — 직접 생성** | 템플릿셋 관리(`/template-sets`) → [생성] → `/template-sets/new`. 빈 폼에서 시작 |

### 폼 구성 (5개 탭)

#### 탭 1. 기본 정보
| 필드 | 값 |
|---|---|
| **템플릿셋명** | 예: "A4 책자 기본 템플릿" |
| **타입** | 책자 (날개+표지+책등+내지) = `BOOK` / 리플렛 (표지+내지) = `LEAFLET` |
| **에디터 모드** | 단일모드 (개별 페이지 편집) = `SINGLE` / 책모드 (스프레드 편집) = `BOOK` |
| **너비/높이 (mm)** | 셋 판형. **표지 한 면 크기**(내지 템플릿 필터와 일치). IDML 인계 시 자동 설정 |

#### 탭 2. 페이지 · 면지
| 필드 | 값 |
|---|---|
| **내지 추가 허용** (`canAddPage`) | 켜면 고객 PDF/페이지 수에 따라 내지 자동 확장 |
| **최소/최대 페이지** | `canAddPage` 켤 때 노출. 페이지 수 범위(`pageCountRange`) |
| **면지 사용** (`useEndpaper`) | 표지 안쪽(앞면지)/뒤표지 안쪽(뒷면지) 빈 페이지 추가 |
| **앞면지/뒷면지 개수** | 각 0~6장 |
| **앞면지/뒷면지 편집 가능** | 면지를 고객이 편집할 수 있는지 |

#### 탭 3. 표지 · PDF 출력
| 필드 | 값 |
|---|---|
| **표지 편집 가능** (`coverEditable`) | 레더 커버/화보집처럼 표지를 사전 인쇄하면 **끄기** → 표지 미리보기 이미지로 대체 |
| **표지 미리보기 이미지** | 레더 커버용. 편집기에선 표지로만 표시, 인쇄 PDF 표지는 빈 페이지 |
| **PDF 첨부 파일 편집 가능** (`contentPdfEditable`) | 끔=첨부 내지 PDF를 가이드로만 표시(편집 불가). 최종 내지는 첨부 원본 PDF 그대로 |
| **PDF 생성 방식** (`pdfOutputMode`) | 단면(1파일·1페이지) / 양면-원파일(앞,뒤,앞,뒤…) = `duplex-merged` / 양면-파일분리 = `duplex-split` ※ 단일·낱장 상품에 적용 |
| **색 처리 방식** (`colorMode`) | RGB 유지(기본) / CMYK 변환(인쇄용) ※ 워커 실제 색변환은 검증 후 라이브 |
| **블리드(사방 mm)** | 재단선 밖 여백. 작업사이즈=재단+블리드×2 |
| **재단선 마커 표기** (`cropMarkEnabled`) | PDF에 코너 재단 마커 |
| **업로드 허용오차(mm)** | 고객 업로드 PDF 사이즈 검증 허용오차 |

| 필드 | 값 |
|---|---|
| **노출할 라이브러리 카테고리** (`libraryCategoryIds`) | 이 셋 편집기에서 노출할 에셋(배경·도형·클립아트·프레임·폰트)을 카테고리 단위로. **비우면 전역(전체 노출)** |

#### 탭 5. 템플릿 구성
- **[템플릿 추가]**(판형 `너비×높이` 필터로 후보 표시) → 셋에 편집할 템플릿들을 추가.
- **드래그로 순서 변경**, 템플릿별 **필수(required) 토글**, 제거.
- **책모드 검증**(실시간):
  - 스프레드 템플릿 **정확히 1개** 필요.
  - 날개/표지/책등 템플릿은 책모드에서 **사용 불가**(스프레드 1 + 내지 PAGE N만).
  - 단일모드에서는 스프레드 템플릿 사용 불가.
- 검증 통과 시 "템플릿 구성이 올바릅니다" 표시.

### 저장
하단 **[생성]**(신규) / **[수정]**(편집). **책모드 저장 검증** = SPREAD **정확히 1개** + 날개/표지/책등 **사용 불가**. ※ 내지(PAGE)는 실제 책 구성에 필요하지만 **폼 저장 시 강제되지 않습니다**(PAGE 0개여도 저장 통과) — 내지는 반드시 추가하세요.

# ③ 템플릿 객체 속성 관리

객체의 **고정 여부 · 편집 권한 · 노출 메뉴**를 관리하는 방법입니다. 변환·편집·셋 단계에 걸쳐 적용됩니다.

### 객체 종류와 식별(extensionType)

| 객체 | extensionType / id | 성격 |
|---|---|---|
| 텍스트 / 이미지 / 요소(SVG·도형) | (없음) | 일반 사용자 객체 — 자유 편집 |
| 배경 색/이미지 | `extensionType: 'background'` | 배경 |
| 프레임 | `extensionType: 'frame'` | 사진 클리핑 창 |
| **템플릿 요소** | `extensionType: 'template-element'` | 관리자 배치 디자인 요소 |
| 워크스페이스 | `id: 'workspace'` | 시스템(선택 제외) |
| 컷/세이프 보더 · 가이드 · 클리핑 마스크 | `extensionType: 'overlay' / 'guideline' / 'clipping'` | 시스템 |

### 1) 배경 아트워크 고정 — `ARTWORK_LOCK`
- **플랫형(hybrid/flat-spine)·PSD 배경 PNG**는 변환기가 `ARTWORK_LOCK`으로 emit합니다:
  - **고정**(`selectable:false`) — 고객이 이동/삭제 불가.
  - **패널 숨김** — 레이어 패널에 노출 안 됨.
  - **관리자 잠금** — 템플릿 요소로 보존.
- 효과: 고객은 **텍스트만 편집**, 배경 디자인은 흔들리지 않음.

### 2) 객체별 편집 권한 — `movable` / `deleteable` (현재 정책: default-permissive)
- 현재 구현은 **default-permissive**(PERM-1):
  - 플래그 미지정(`undefined`) = **허용**(이동/삭제 가능).
  - 관리자가 **'위치 고정'으로 명시 잠근 객체만** 잠김.
- 이유: 'default-locked'를 적용하면 플래그 없는 **기존 라이브 템플릿·주문 전체 객체가 일괄 동결**되는 회귀가 생기므로, 1차는 admin opt-in(명시 잠금)으로 출하.
- 설계 정본: `.cursor/plans/PHOTO_FRAME_UX_AND_OBJECT_LOCK_DESIGN_2026-06-16.md` §4(Part B)·§7.

### 3) 템플릿/면지/표지 단위 편집 가능 토글
| 토글 | 위치 | 효과 |
|---|---|---|
| 템플릿 **editable** | 템플릿 | readonly 처리(객체 추가/이동 차단) |
| **표지 편집 가능**(`coverEditable`) | 템플릿셋 탭3 | 끄면 표지 미리보기 이미지로 대체(레더 커버) |
| **면지 편집 가능**(앞/뒤) | 템플릿셋 탭2 | 면지 편집 허용 여부 |
| **PDF 첨부 편집 가능**(`contentPdfEditable`) | 템플릿셋 탭3 | 끄면 첨부 내지 PDF를 가이드로만 |

### 4) 도구 메뉴 화이트리스트 (셋 단위)
- 템플릿셋 탭4 **도구 메뉴 노출 직접 설정** → `enabledMenus`로 노출할 좌측 메뉴를 한정.
- 빌드 플래그(`requiresFlag`)가 꺼진 메뉴는 화이트리스트와 무관하게 숨겨집니다.

> ⚠️ 2026-06-11 이전 변환본은 재편집 시 객체 투명/붕괴·좌표 변위 결함이 있을 수 있습니다(`527b85b`에서 수정). 해당 등록본은 **재가져오기** 필요. 신규 가져오기는 정상.

# ④ 상품-템플릿셋 연결 방법

북모아 상품코드(sortcode)에 템플릿셋을 매핑합니다. 고객이 해당 상품을 주문하면 연결된 템플릿셋이 자동 제공됩니다(1:N).

### 진입
admin 좌측 메뉴 **상품-템플릿셋 연결 관리**(`/product-template-sets`).

### 데이터 모델 (`ProductTemplateSet`)
| 컬럼 | 뜻 |
|---|---|
| `sortcode` | 북모아 카테고리(상품) 코드 (예: `001001001`) |
| `prdtStanSeqno` | 북모아 규격 번호. **NULL이면 해당 상품의 모든 규격에 적용** |
| `templateSetId` | 연결할 템플릿셋 |
| `displayOrder` | 표시 순서 |
| `isDefault` | 기본 템플릿(고객에게 먼저 추천) |
| `isActive` | 활성 여부 |

### 연결 단계
1. **[연결 추가]** 클릭 → "상품-템플릿셋 연결 추가" 모달.
2. **상품코드 (sortcode)**: 자동완성 입력(상품명/코드로 북모아 카테고리 검색, 2자 이상).
3. **규격 번호 (선택)**: 비우면 해당 상품 모든 규격에 적용.
4. **템플릿셋 선택**(다중): 검색·다중 선택. 셋 정보(책자/리플렛 · 판형)가 라벨에 표시.
5. **기본 템플릿으로 설정**(1개 선택 시 토글). 여러 개 선택 시 **순서대로 `displayOrder` 부여, 첫 번째가 기본**.
6. **[추가]** → 1개=단건 생성, 여러 개=일괄 생성.

### 목록 관리
- **기본(★)**: 별 아이콘 토글로 기본 템플릿 지정.
- **순서(displayOrder)**: 정렬 컬럼.
- **활성(스위치)**: 노출 on/off.
- **면지/레더커버 배지**: 템플릿셋 요약(예: `면지 앞2/뒤2`, `레더커버`) 표시.
- **삭제**: 연결 제거(템플릿셋 자체는 유지).

### 외부 조회 (북모아 연동)
- 외부 조회 API: `GET /api/product-template-sets/by-product?sortcode=<코드>&stanSeqno=<규격>`
- ⚠️ 이 외부 엔드포인트는 **API Key 인증 필요**(`@Public` + `ApiKeyGuard`, `X-API-Key` 헤더 / `STORIGE_API_KEY`). 관리자용 목록·생성·수정·삭제는 Bearer 토큰 + RolesGuard.
- **폴백 조회 로직**:
  1. `sortcode` + `prdtStanSeqno` 정확 매칭.
  2. 없으면 `sortcode`만(규격 NULL 행).
  3. `displayOrder` 순 정렬.
  4. `isDefault=true`를 첫 번째로.
  5. **소프트 삭제(isDeleted)된 템플릿셋과 연결된 행은 결과에서 제외**(total도 제외 후 카운트).
- 북모아 진입(`bookmoa/front/storige/edit.php`)에서 `storige_get_template_sets($sortcode, $stan_seqno)`로 목록을 받아 에디터에 전달.

## 부록 C. 현재 유형 인벤토리 + 새 상품 유형 추가 지점 (2026-06-23 코드 대조)

> 2026-06-23 본문을 코드(`packages/types/src/index.ts`)와 전수 대조 — **본문 유형/모드 기술은 현 코드와 일치**(드리프트 없음). 새 상품 유형 추가를 위한 확장 지점을 정리한다.

### C-1. 현재 유형/모드 전수 (코드 정본)

| 축 | 값 | 타입 | 위치(types/src/index.ts) |
|---|---|---|---|
| 템플릿 타입 `TemplateType` | wing · cover · spine · page · spread · endpaper (6) | enum | `:79` |
| **상품 유형(템플릿셋) `TemplateSetType`** | **book · leaflet (2)** | enum | `:94` |
| 에디터 모드 `EditorMode` | single · book (2) | enum | `:1304` |
| IDML 변환 모드 `SpreadConversionMode` | full · flat-spread · flat-spine (3) | union | `:1513` |
| PDF 출력 모드 `PdfOutputMode` | single · duplex-merged · duplex-split (3) | union | `:262` |
| 합성 출력 모드 `ComposeOutputMode` | separate · content-only · single (3) | union | `:674` |
| 색 모드 `colorMode` | rgb · cmyk (2) | union | (TemplateSetConfig) |

> 부수: `TemplateSetConfig`에 `productSpecs?`(상품 규격 메타) 필드 존재 — 본문 탭에 미노출(내부/연동용).

### C-2. "상품 유형"의 실제 모델링 (enum ≠ 사용자 상품 1:1)

현 사용자 관점 상품(명함·책자·포스터·엽서·레더커버 등)은 **소수 enum의 조합 + 템플릿셋 설정**으로 표현된다:

| 사용자 상품 | 모델링 조합 |
|---|---|
| 책자(양장/무선) | `BOOK` + EditorMode `book` + spread 1 + page N (+ 면지) |
| 리플렛/낱장(명함·엽서·카드) | `LEAFLET` + EditorMode `single` + page + `pdfOutputMode`(single/duplex) |
| 레더커버/화보집 | `BOOK`/`LEAFLET` + `coverEditable=false` + `coverPreviewImage` |
| 포스터/단면 | `LEAFLET` + `pdfOutputMode=single` |

→ **새 상품 유형은 대개 enum 추가 없이** 기존 타입 조합 + 셋 설정으로 가능. 진짜 새 카테고리(예: 캘린더·박스/패키지·아코디언 접지 등 출력 규칙 자체가 다른 것)만 enum 확장이 필요.
