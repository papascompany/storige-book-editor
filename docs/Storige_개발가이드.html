Step-by-Step 실행 순서 (경로 b)
Step A — init.sql 백업 cp docker/mysql/init.sql docker/mysql/init.sql.bak.$(date +%Y%m%d)
Step B — P0A_DB_SCHEMA_FIX.md의 완성된 SQL로 init.sql 전체 교체 23개 테이블 DDL 포함. admin 비밀번호 bcrypt 해시 생성 후 주입.
Step C — MariaDB 볼륨 초기화 후 재기동 docker-compose down -v && docker-compose up -d mariadb
Step D — 테이블 수 검증 → 23개 확인 docker exec -it storige-mariadb mysql -u storige -p storige -e "SHOW TABLES;"
Step E — API 기동 및 ER_NO_SUCH_TABLE 에러 없음 확인 GET http://localhost:4000/api/edit-sessions → HTTP 200 (빈 배열 OK)

자주 묻는 Q&A
Q. pnpm dev가 types를 못 찾아요
A. pnpm --filter @storige/types build를 먼저 실행. canvas-core도 함께 빌드: pnpm --filter @storige/canvas-core build
Q. Worker는 도는데 상태가 API에 반영 안 됩니다
A. apps/worker/.env에 WORKER_API_KEY, API_BASE_URL이 있는지 확인. 둘 다 없으면 조용히 실패. 추가로 WORKER_API_KEY가 루트 API_KEYS 목록에 포함되어 있어야 함.
Q. 관리자 페이지 401 무한 루프
A. 브라우저 localStorage에서 accessToken/refreshToken 삭제 후 재로그인. 에디터와 동시 사용 시 세션 충돌 → 시크릿 창 또는 별도 브라우저 프로필 사용.
Q. 편집 후 PDF 저장이 중간에 멈춥니다
A. useWorkSave.ts P1 이슈 가능성 높음. 네트워크 탭에서 /edit-sessions/:id/complete가 호출되는지 먼저 확인.
Q. Docker worker 컨테이너가 canvas 빌드 실패
A. docker/worker/Dockerfile에 cairo/pango/ghostscript/imagemagick 설치 필요. 오래된 이미지면 --no-cache 재빌드. Apple Silicon은 --platform linux/amd64 명시.
Q. .env와 Vercel 환경변수가 따로 놀아요
A. storige-admin .env.production이 Vercel 값을 override하는 구조. 커밋 012e1e9, f2bab8d 참고.
Q. Bookmoa DB 연결 오류
A. BOOKMOA_DB_*는 기본 주석 처리. 필요할 때만 .env에서 해제. 반드시 읽기 전용 계정 사용.
