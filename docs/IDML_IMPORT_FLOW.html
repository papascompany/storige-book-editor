2 시나리오별 상세

A. 펼침면 단품 (하이브리드 "이미지로 누르기")

1) 업로드 → 하이브리드 → 텍스트만 편집 + 배경 PNG
2) 미리보기 확인
3) 등록 = 표지 단품 → POST /templates
4) 같은 사이즈 다른 디자인 추가 → 편집기 '템플릿'에서 교체
5) 텍스트 미세조정 → 저장 → 인쇄 PDF 확인

B. 책등 가변 템플릿셋

1) 업로드 → 벡터(또는 하이브리드) → 등록 = 책등 가변 셋
2) 표지 저장 → 셋 폼 자동 인계(표지+책모드+판형)
3) 수동: 내지 추가 + pageCountRange + PDF/색 모드
4) 셋 저장 → 상품 연결
5) 핵심: 내지 수 변경 → 책등 폭 자동 재계산 + 앞/뒤표지 평행이동, 재로드 유지

3 책등 가변 원리 (왜 자동인가)

책등 폭은 템플릿 고정값이 아니라 런타임 파생값: spineWidth = (pageCount/2) × thickness + margin.
권위 검증은 coverWidth/coverHeight/wingEnabled/wingWidth 4필드만 비교(책등/총폭은 제외).
→ IDML 원본의 고정 책등을 버리고 표지 한 면+날개만 권위로 추출하면 자연 가변화. 책등 텍스트=중앙 재배치, 앞/뒤표지=평행이동(리플로우 손실 없음).

4 검증 체크리스트

변환 충실도

  • z-순서(쌓임) 원본대로
  • 책등 제목/저자 세로글씨(90°)
  • 여러 줄 텍스트 줄바꿈 보존(영역 이탈 없음)
  • 폴리곤/곡선 베지어 경로 복원
  • mm 치수 정확(총폭=날개+앞+책등+뒤+날개)

인쇄 규약 (신규)

  • 색 처리: 셋 colorMode대로(RGB/CMYK)
  • 텍스트: 인쇄 PDF 아웃라인 벡터, 혼합폰트 각 폰트
  • overprint/별색: 고객 PDF /OP·Separation 보존
  • 프레임: 사진이 투명창에만, 밖은 가림

책등 가변 / 워크플로우

  • 내지 수 변경 → 책등 폭 자동 재계산 + 앞/뒤표지 평행이동
  • 편집완료→재로드 후 책등 가변 유지(meta 보존)
  • 같은 사이즈 다른 표지로 교체(편집기 '템플릿')

6 문제 해결

증상점검
변환 텍스트 폰트 다름/깨짐폰트 라이브러리 시딩 여부. 미임베드는 폴백(base 폰트/래스터)
객체 영역 밖 이탈미리보기 확인 → 편집기 템플릿편집 모드로 미세조정(전 객체 selectable)
책등이 안 변함책모드 셋인지, SPREAD 1개+PAGE≥1개인지, meta(regionRef) 보존됐는지
책 셋 저장 안 됨book 검증=SPREAD 정확히 1개 필수 → 내지(PAGE) 추가했는지
표지 색이 화면과 다름RGB 파이프라인 색차. 셋 colorMode=cmyk + 워커 색변환(적용 후)
프레임에 사진 삐져나옴/반대로 가림프레임 마스킹 방향(inverted) 검증 — 반대면 보고
변환 PNG 안 보임스토리지 URL(/storage/...) 접근성(CORS) 확인
