1 · 시스템 관계도

PHP가 X-API-Key로 8개 외부 엔드포인트를 호출하고, Storige가 callbackUrl로 웹훅을 회신합니다. Editor/Admin은 Vercel CDN에서 정적 서빙되며 API 호출은 직접 api.papascompany.co.kr로.

2 · 모듈 의존성

의존성 핵심 규칙

  • @storige/types는 항상 먼저 빌드 (다른 모든 곳이 import). Dockerfile에서도 builder → production stage로 packages/types/dist 명시 복사.
  • Vercel 빌드 시 --filter @storige/{editor,admin}...로 worker 의존성 install 회피.
  • jsdom의 옵셔널 canvas는 pnpm.ignoredOptionalDependencies로 무시 (Vercel native 빌드 차단).
  • Worker(Docker)는 canvas 정식 의존, builder에 cairo/pango/jpeg/giflib/librsvg-dev 설치 → 컴파일 → 삭제.

7 · PHP 웹훅 검증 (Storige → PHP)

요청 헤더

Content-Type: application/json
X-Storige-Event:     session.validated  // or session.failed | synthesis.completed | synthesis.failed
X-Storige-Signature: <base64>
X-Storige-Retry:     1                  // 있으면 재시도분

시그니처 알고리즘 (현재)

signature = base64( identifier + ":" + event + ":" + timestamp )
identifier = payload.sessionId 또는 payload.jobId.

PHP 검증 헬퍼

function verifyStorigeSignature(string $rawBody, string $headerSig): bool {
  $payload = json_decode($rawBody, true);
  $id = $payload['sessionId'] ?? $payload['jobId'] ?? 'unknown';
  $expected = base64_encode("{$id}:{$payload['event']}:{$payload['timestamp']}");
  return hash_equals($expected, $headerSig);
}
⚠ 강도 한계 현재 시그니처는 비밀키 없는 base64 — 위변조 가능. P6에서 WEBHOOK_SECRET HMAC SHA256 도입 권장.

4가지 페이로드 분기

event의미핵심 필드
session.validated검증 성공sessionId, orderSeqno, fileType, result
session.failed검증 실패sessionId, errorMessage
synthesis.completed합성 성공jobId, sessionId, mode, outputFileUrl
synthesis.failed합성 실패jobId, errorMessage

핸들러 골격

// /webhook/storige.php
$rawBody = file_get_contents('php://input');
$sig = $_SERVER['HTTP_X_STORIGE_SIGNATURE'] ?? '';
if (!verifyStorigeSignature($rawBody, $sig)) {
    http_response_code(401); exit('bad signature');
}
$p = json_decode($rawBody, true);
switch ($p['event']) {
  case 'session.validated':   /* DB 갱신 */ break;
  case 'session.failed':      /* 알림 */     break;
  case 'synthesis.completed': /* outputFileUrl 저장 */ break;
  case 'synthesis.failed':    /* 재시도 / 알림 */      break;
}
http_response_code(200); echo 'ok';

8 · Editor 임베드 (PHP 페이지)

<!-- bookmoa의 편집 페이지 -->
<iframe id="storige-editor"
        src="<?= STORIGE_EDITOR_URL ?>/?templateSetId=<?= $setId ?>&orderSeqno=<?= $orderNo ?>"
        style="width:100%;height:100vh;border:0;"></iframe>

<script>
  window.addEventListener('message', (e) => {
    if (e.origin !== 'https://editor.papascompany.co.kr') return;
    const { type, payload } = e.data || {};
    switch (type) {
      case 'storige:saved':     /* 자동 저장 */ break;
      case 'storige:completed': /* sessionId 받음 */ break;
      case 'storige:error':     /* 에러 */ break;
    }
  });
</script>
쿠키 정책 크로스 사이트 iframe 인증을 위해 Storige API가 발급하는 쿠키는 반드시 SameSite=None; Secure; HttpOnly. PHP 운영 도메인이 우리와 다른 서브도메인이면 반드시 운영 시작 전 점검.
CORS_ORIGIN ~/storige/.env의 CORS_ORIGIN에 PHP 운영 도메인도 반드시 포함. 누락 시 iframe → API 호출이 차단됨.
