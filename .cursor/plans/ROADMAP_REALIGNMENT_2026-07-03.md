## 2. "잘한 결정이었나" — 정직한 판정

**지금까지 한 것은 잘한 결정이다. 그러나 여기서 더 가면 잘못된 결정이 된다.**

- 완료된 "확장" 작업의 실체는 확장이 아니라 **안전판+성능**이었다: Track A는 성능 안정화(42%↓, 출력 불변 실증), Phase 0은 회귀 그물(CI·contract test·골든하네스)과 **실존 보안 구멍 봉합**(thumbnail 무인증 크로스테넌트 유출). 이건 어느 방향으로 가든 필요했던 투자이고, 되돌릴 것도 후회할 것도 없다.
- **진짜 갈림길은 지금이다.** Phase 1(SDK·OpenAPI·PrintCard 파이프라인)부터가 "수요를 가정한 확장 베팅"이다. 그런데 Phase 4 후보 3곳 전부 **수요 확약이 없다**. 1인 운영에서 11~20주짜리 베팅을 확약 없이 시작하는 것이 유일하게 위험한 선택지다.
- 로드맵이 "엉켰다"는 느낌의 원인: Phase 0~6이 직렬로 보이지만 실제로는 **성격이 다른 세 트랙**(코어 제품 / 플랫폼 확장 / 보안·유지)이 한 줄에 섞여 있었기 때문. 아래처럼 트랙을 분리하고 확장을 트리거 기반으로 바꾸면 풀린다.

## 3. 목표 명확화 (제안 — 북극성)

> **"Storige = Bookmoa 인쇄 주문을 무사고로 완결하는 편집기+PDF 파이프라인이 코어.
> 그 파이프라인을 '검증된 수요가 있는' 파트너에게만 임대한다."**

- 코어 지표: 주문 완결률(편집→PDF→발주 무사고) · 템플릿 제작 속도(오너 운영 효율) · 편집 UX 이탈
- 확장 지표: **파트너 수요 확약(계약/LOI) 건수** — 이것이 0이면 확장 코드를 늘리지 않는다
- 이미 검증된 원칙의 일반화: "트랙 B는 4번째 상품 트리거" → **모든 확장 Phase에 트리거를 붙인다**

## 4. 재정의 로드맵 — 3트랙 + 트리거

### 트랙 C (Core — **지금 재개**, 4~6주)
1. **C-1 편집기 UX 묶음** (오너 체감 최대): 라이브러리 객체 컨트롤(추가/제거/이동) · **템플릿 생성 모드 레이어별 속성 지정**(lockInfo/deleteable/movable 권한모델 기존재 — UI 배선) · 사진틀 스왑 발견성. ⚠️ 공유 UX 원칙 유지(TemplateSetType 게이팅 0건) + 5개 라이브러리 static import 재유입 금지.
2. **C-2 워커 처리조건 복귀**: crop marks 실배선 · DEFAULT_* 상수 소비 연결 · lightweight-synthesis 프로덕션 ON 검토 · stale 주석 정리. (워커는 Track A와 충돌 0 — 병행 안전 확정됨)
3. **C-3 잔손질**: 편집기 누수 2건(지금이 적기) · retention dryRun 해제(근거 완비 — 오너 승인만).
4. **C-4 포토북 O-2 잔여** — 포토북 출시 일정에 종속(오너의 상품 우선순위 결정 대기).

### 트랙 P (Platform — **트리거 대기** + 저비용 정보수집만)
- **지금 해도 되는 것(저비용·고정보)**: ① PrintCard CMYK/N-up worker 실사(1~2일, 스코프 확정만 — 코드 작성 아님) ② Sharesnap 언블록(회신 오면 몇 시간).
- **트리거 A — 파트너 확약 1건 발생 시**: Phase 1(SDK 초판, 그 파트너 시나리오로 범위 재단) → Phase 2(웹훅 v2 opt-in, 그 파트너 1호).
- **트리거 B — 4번째 편집상품 착수 시**: 트랙 B Capability Registry.
- **트리거 C — 파트너 3곳 이상 활성 시**: Phase 3 통합 admin(그 전엔 런북+SQL로 충분).
- 확장을 멈춰도 자산이 부패하지 않는 이유: **CI + contract test + 대조표 + 골든하네스가 계약을 지킨다**(이번에 만든 것의 진짜 가치).

### 트랙 S (Security/유지 — 상시 소량)
- P1 workerAuthCode 교차테넌트(파트너 늘기 전 = 트리거 A 진입 시 필수 선결) · bookmoa PHP 구 키 cutover(보류 유지) · git history force-push · admin 쿠키 stage1b · Bull attempts — 전부 오너 게이트 항목, 월 1회 점검.
